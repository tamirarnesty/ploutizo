# Sketch 005: Card balances subtitle legend

## Design Question

How should the `Card Balances` subtitle render top-3 member totals (+ overflow), with color coding matched to Split by member badges, and what icon treatment should we use for settle/status/brand in the final design?

## How to View

Open `.planning/sketches/005-card-balances-subtitle-legend/index.html`.

## Variants

- **A: Plain subtitle text** — readable text line with color dots and separators.
- **B: Badge subtitle row** — component-like badge strip, stronger chip affordance.
- **C: Compact token subtitle** — low-noise token list with separators.

## What to Look For

- Which subtitle feels informative without adding visual heaviness.
- Whether the top-3 + overflow treatment is enough context.
- Which icon strategy (settle/status/brand) reads fastest while staying accessible.
- Whether the overall result still feels like your chosen C baseline.

## Problem

Navigating to an unknown route shows a blank screen or default browser error. TanStack Start supports a `notFoundComponent` on the root route that renders when no route matches.

## Solution

Implement a `notFoundComponent` on the root route (and optionally per-route `notFoundComponent` for nested misses). The page should display a friendly "Page not found" message with a link back to the dashboard.

## Summary

Phase 03.5 added a `format:check` turbo task, package-level `format:check` scripts in `apps/web` and `packages/ui`, and a four-job GitHub Actions CI workflow. The workflow structure, permissions, concurrency, and action versions are correct. One coverage gap was found: four packages in the monorepo (`apps/api`, `packages/db`, `packages/types`, `packages/validators`) are missing a `format:check` script, so `pnpm turbo format:check` silently skips their source files. Two info-level items are noted.

## Warnings

### WR-01: `format:check` skips four packages â€” false coverage confidence

**Files:** `apps/api/package.json`, `packages/db/package.json`, `packages/types/package.json`, `packages/validators/package.json`

**Issue:** Turbo dispatches `format:check` only to packages that declare that script. `apps/api`, `packages/db`, `packages/types`, and `packages/validators` all contain TypeScript source but have no `format:check` entry in their `scripts`. `pnpm turbo format:check` completes without error and without checking those packages. Unformatted code in those packages will pass CI.

**Fix:** Add `format:check` to all four packages (same pattern already used in `apps/web` and `packages/ui`):

```json
"format:check": "prettier --check \"**/*.{ts,tsx}\""
```

Alternatively, add a root-level `format:check` script that invokes prettier directly across the whole repo rather than routing through turbo:

```json
"format:check": "prettier --check \"**/*.{ts,tsx}\" --ignore-path .gitignore"
```

## Info

### IN-01: GitHub Actions pinned to major version tags, not commit SHAs

**File:** `.github/workflows/ci.yml:22,25,30,49,52,57,73,76,81,97,100,105`

**Issue:** `actions/checkout@v6`, `actions/setup-node@v6`, and `pnpm/action-setup@v6` are pinned to mutable major-version tags. A compromised or accidental tag re-point would silently run different code in CI without a diff in this repo. SHA pinning (e.g. `actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683`) is the supply-chain-safe alternative recommended by GitHub's hardening guide.

**Fix:** Pin each action to its full commit SHA and add a comment with the resolved version for readability. Use Dependabot or Renovate with the `github-actions` ecosystem to receive automated SHA-bump PRs.

### IN-02: Root `package.json` has no `format:check` script

**File:** `package.json` (root)

**Issue:** The root `scripts` block exposes `lint`, `format`, `typecheck`, `test`, and `build` as top-level conveniences, but not `format:check`. Developers running `pnpm format:check` from the repo root get `ERR_PNPM_NO_SCRIPT`. The CI workflow runs `pnpm turbo format:check` directly, so CI itself is unaffected, but the missing root script is inconsistent with the pattern of all other turbo tasks.

**Fix:**

```json
"format:check": "turbo format:check"
```

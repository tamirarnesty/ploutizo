  <files>apps/web/package.json, packages/ui/package.json</files>
  <read_first>
    - apps/web/package.json — read the `scripts` block to see the existing `format` script glob before adding `format:check`
    - packages/ui/package.json — same
  </read_first>
  <action>
In both `apps/web/package.json` and `packages/ui/package.json`, add a `format:check` script immediately after the `format` script entry.

Both packages have the same `format` script: `prettier --write "**/*.{ts,tsx}" && eslint --fix`

The `format:check` script replaces `--write` with `--check` and drops the `eslint --fix` (ESLint has its own CI job):

```json
"format:check": "prettier --check \"**/*.{ts,tsx}\""
```

Place the entry after the `format` line in the scripts object in both files.

The glob `"**/*.{ts,tsx}"` matches the same files as the write script but will be filtered by the root `.prettierignore` (which excludes dist/, node_modules/, .turbo/, .output/, .nitro/, .tanstack/, .vinxi/, coverage/).

Do NOT add a `format:check` script to packages that have no `format` script (apps/api, packages/db, packages/types, packages/validators) — turbo skips packages that don't define a task, which is correct behavior.
  </action>

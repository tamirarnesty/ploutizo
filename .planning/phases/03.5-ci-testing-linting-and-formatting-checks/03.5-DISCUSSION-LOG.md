## Format check

| Option | Description | Selected |
|--------|-------------|----------|
| Add format:check scripts | Add `format:check` turbo task + `prettier --check` scripts to each package. Runnable locally and in CI the same way. | ✓ |
| Workflow-only prettier --check | Call `prettier --check .` directly in the workflow. No new scripts needed, but check isn't runnable locally the same way. | |

**User's choice:** Add format:check scripts (Recommended)

---

## Job structure

| Option | Description | Selected |
|--------|-------------|----------|
| Parallel jobs | Separate jobs for lint+typecheck, test, and format-check. Faster PR feedback — each leg runs concurrently. | ✓ |
| Single job | All steps sequential in one job. Simpler YAML, pnpm install once. A lint failure blocks test output. | |

**User's choice:** Parallel jobs (Recommended)

---

## Build in CI

| Option | Description | Selected |
|--------|-------------|----------|
| Skip build | TypeScript typecheck catches most type errors. Build adds 2-3 min per run and is heavy for a hobby project. | |
| Include build job | Catches bundler-level errors (bad imports, vite config issues) that tsc --noEmit doesn't surface. | ✓ |

**User's choice:** Include build job

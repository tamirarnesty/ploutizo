<decisions>
## Implementation Decisions

### Format check
- **D-01:** Add a `format:check` turbo task and `prettier --check` script to each relevant package (apps/web, packages/ui — the two that currently have `format` scripts using `prettier --write`). The root workflow calls `pnpm turbo format:check`. This makes the check runnable locally the same way as in CI.

### Job structure
- **D-02:** Parallel jobs. Separate concurrent jobs for: lint + typecheck, tests, format-check, and build. Faster PR feedback — each leg runs independently and concurrently.

### Build in CI
- **D-03:** Include a build job. Runs `pnpm turbo build` to catch bundler-level errors (bad imports, vite config issues) that `tsc --noEmit` doesn't surface.

### Trigger
- **D-04:** CI triggers on `pull_request` (all branches) and `push` to `main`. Standard pre-merge and post-merge coverage.

### Claude's Discretion
- pnpm version pinning in workflow (use packageManager field from root package.json: pnpm@9.15.9)
- Node version pinning (use .node-version: 22.14.0)
- Turbo remote cache: skip for now (no TURBO_TOKEN setup needed)
- Concurrency groups to cancel stale runs on the same PR
- Job naming and step labels

</decisions>

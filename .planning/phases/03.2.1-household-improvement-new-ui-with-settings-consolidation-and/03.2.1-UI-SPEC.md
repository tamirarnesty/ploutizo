## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge gaps |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, member row padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Member list rows: `px-4 py-3` (16px / 12px) — matches existing `OrganizationMembersSettings` pattern
- Avatar in member rows: `size-8` (32px) — sm Avatar variant
- Separator between sections: 0px margin (sections use `gap-8` on parent `flex-col`)
- Touch target minimum: 44px height for remove buttons on member rows

Source: D-01 (CONTEXT.md) — `flex flex-col gap-8 max-w-2xl`; existing `OrganizationMembersSettings` row pattern

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 600 (semibold) | 1.25 |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |
| Caption / muted | 12px (`text-xs`) | 400 (regular) | 1.5 |

Notes:
- Section headings (`h2`) use `text-sm font-semibold` — two-weight system: 400 for body/caption, 600 for labels and headings
- Page-level heading (`h1`) is NOT rendered in the settings page body — the settings layout renders "Settings" at the route level; the component renders section headings only
- `font-heading` maps to `Noto Serif Variable` — use only on the section title level heading if any (`text-xl font-semibold`), consistent with existing `HouseholdSettings` and `OrganizationMembersSettings` h1 pattern

Source: existing `HouseholdSettings.tsx`, `OrganizationMembersSettings.tsx`, `globals.css`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 light / oklch 0.147 dark) | Page background, settings content area |
| Secondary (30%) | `--card` / `--muted` (oklch 0.97 light / oklch 0.216 dark) | Member list row borders, skeleton loaders, muted descriptions |
| Accent (10%) | `--primary` (oklch 0.5 0.134 242.749 light — blue) | "Send invite" button (primary CTA only) |
| Destructive | `--destructive` (oklch 0.577 0.245 27.325 light) | Remove member button, AlertDialog destructive action button |

Accent reserved for: the "Send invite" submit button only. All other buttons use `variant="ghost"` or `variant="outline"`.

Additional semantic colors in use:
- `--muted-foreground`: helper text, role labels, section descriptions
- `--success` (`--color-emerald-500`): not used in this phase
- Member row borders: `border-border` (`--border`) — explicit token, never bare `border`

## Interaction Contracts

### Household Overview Section
- Renders org `<Avatar>` (`size-10`, 40px) with `imageUrl` and initials fallback
- Org name rendered as `text-sm font-semibold` beside the avatar
- Loading state: single skeleton row `h-10 w-48 animate-pulse rounded bg-muted` with `motion-safe:animate-pulse`
- No edit controls — entirely read-only

Source: D-04, D-05 (CONTEXT.md); Claude's Discretion (avatar size)

### Member Row
- Layout: `flex items-center justify-between rounded-md border-border border px-4 py-3 text-sm`
- Left: `<Avatar size-8>` + display name (`font-semibold truncate min-w-0`) + optional "You" `<Badge variant="secondary">`
- Right: role label (`text-xs text-muted-foreground capitalize`) + remove `<Button variant="ghost" size="icon" aria-label="Remove [name]">`
- Remove button hidden (not just disabled) on the current user's own row
- Loading skeleton: 3 rows of `h-12 animate-pulse rounded bg-muted motion-safe:animate-pulse`
- Empty state: `<p className="text-sm text-muted-foreground">This household has no other members.</p>`

Source: D-06, D-07 (CONTEXT.md); existing `OrganizationMembersSettings.tsx`

### Remove Member Confirmation Dialog
- `<AlertDialog>` triggered by remove button click
- Title: "Remove [First Name]?"
- Body: "Remove [Full Name] from this household? They will lose access immediately."
- Buttons: `<AlertDialogCancel>Keep member</AlertDialogCancel>` + `<AlertDialogAction className="bg-destructive text-destructive-foreground hover:bg-destructive/90">Remove</AlertDialogAction>`
- Dialog width: `w-[calc(100%-2rem)] max-w-md` (mobile pattern from 02.4.1 D-09)
- On confirm: fires `DELETE /api/households/members/:memberId`, shows error toast on failure

Source: D-08 (CONTEXT.md); 02.4.1 D-09 dialog mobile width pattern

### Invite Form
- Inline below member list, no dialog
- Single email `<Input>` with `autocomplete="email"`, placeholder "Email address", full width
- `<Button type="submit">Send invite</Button>` (default/primary variant), positioned to the right of the input or below on mobile
- Inline field validation error rendered below input via `form.AppField` error display (Zod email validation)
- Submit disabled while mutation is in-flight
- On success: input resets to empty, toast shown
- On error: toast shown (no inline error beyond Zod field validation)

Source: D-10, D-11, D-12, D-13 (CONTEXT.md)

## Accessibility Requirements

- All icon-only buttons must have `aria-label` — remove button: `aria-label="Remove [name]"`
- Invite email input: `autocomplete="email"`, `type="email"`, associated `<Label>` via `htmlFor` + `id`
- Focus states: all interactive elements use `ring-ring ring-ring/50` — never `outline: none` without custom focus style
- Animations: `motion-safe:animate-pulse` on all skeleton loaders (already used in existing `OrganizationMembersSettings`)
- `min-w-0` on flex children that render truncated text (member display name)
- `AlertDialog` manages focus trap automatically via Radix — no additional `autoFocus` needed
- Remove button disabled state: hidden entirely for self (`user?.id === member.externalId`) — not just `disabled` attr

Source: CLAUDE.md web-design-guidelines skill; existing codebase patterns

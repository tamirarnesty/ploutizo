### Anti-Patterns Found

| File | Pattern | Severity | Impact |
|------|---------|----------|--------|
| `packages/ui/src/components/skeleton.tsx` | `animate-pulse` without `motion-safe:` prefix | ⚠️ Warning | Users with `prefers-reduced-motion` will still see the pulse animation. The plan specified `motion-safe:animate-pulse` on all skeletons, but the implementation delegates to `<Skeleton>` which uses plain `animate-pulse`. The summary incorrectly claims this was applied. |

**Note on skeleton anti-pattern:** The web-design guidelines require `prefers-reduced-motion` respect for animations. The `<Skeleton>` base component in `packages/ui/src/components/skeleton.tsx` does not apply `motion-safe:`. However, per CLAUDE.md, base components (`packages/ui/src/components/`) must not be modified — the fix would need to be applied at usage sites via className override: `<Skeleton className="motion-safe:animate-pulse [animation:none]" />`. This was not done. Classified as Warning (accessibility gap, not a functional blocker).

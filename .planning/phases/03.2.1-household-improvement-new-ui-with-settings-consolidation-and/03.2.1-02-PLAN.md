    ```typescript
    householdsRouter.delete('/members/:memberId', async (c) => {
      const { orgId, userId: callerClerkId } = getAuth(c);
      const { memberId } = c.req.param();

      // Look up local member row to get externalId for Clerk call (D-18)
      const member = await db
        .select({ externalId: users.externalId })
        .from(orgMembers)
        .innerJoin(users, eq(users.id, orgMembers.userId))
        .where(and(eq(orgMembers.id, memberId), eq(orgMembers.orgId, orgId!)))
        .then((r) => r.at(0));

      if (!member) {
        return c.json({ error: { code: 'NOT_FOUND' } }, 404);
      }

      // Server-side self-removal guard (D-19)
      if (member.externalId === callerClerkId) {
        return c.json({ error: { code: 'SELF_REMOVAL_FORBIDDEN' } }, 403);
      }

      // Call Clerk API to remove membership (D-17)
      await fetch(
        `https://api.clerk.com/v1/organizations/${orgId}/memberships/${member.externalId}`,
        {
          method: 'DELETE',
          headers: { Authorization: `Bearer ${process.env.CLERK_SECRET_KEY}` },
        }
      );

      // Remove local org_members row (D-17)
      await db.delete(orgMembers).where(eq(orgMembers.id, memberId));

      return c.json({ data: { removed: true } });
    });
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (POST /invitations) | Email address from request body sent to Clerk REST API |
| client→API (DELETE /members/:memberId) | memberId path param used to look up and delete org member |
| API→Clerk REST | Server-side calls using CLERK_SECRET_KEY — must never reach client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.2.1-02-01 | Elevation of Privilege | DELETE /members/:memberId self-removal | mitigate | Server-side guard: `if (member.externalId === callerClerkId) return 403` (D-19). Client-side hidden button is not sufficient alone. |
| T-03.2.1-02-02 | Tampering | DELETE /members/:memberId IDOR | mitigate | Query includes `.where(and(eq(orgMembers.id, memberId), eq(orgMembers.orgId, orgId!)))` — org-scoping prevents cross-org member deletion |
| T-03.2.1-02-03 | Information Disclosure | CLERK_SECRET_KEY | mitigate | Used only via `process.env.CLERK_SECRET_KEY` in server-side Hono routes. Never referenced in apps/web. |
| T-03.2.1-02-04 | Spoofing | POST /invitations — email injection | accept | Email field passed directly to Clerk API which validates email_address format. Clerk returns 422 on invalid email; app maps to 400. Zod validation at form layer provides additional defense. Low residual risk. |
</threat_model>

# Phase 02.1 Plan 03: CategoryDialog + RuleDialog Form Split Summary

**One-liner:** CategoryDialog and RuleDialog refactored into chrome/form pairs using TanStack Form, with LucideIconPicker/ColourPicker custom input binding and null categoryId eliminating the __none__ sentinel.

## What Was Built

### Task 1: Split CategoryDialog into chrome + CategoryForm

- **CategoryDialog.tsx** rewritten to ~18 lines: Dialog chrome only, imports and renders `<CategoryForm />`, no useState, no form JSX
- **CategoryForm.tsx** created: TanStack Form with `useAppForm`, `zodValidator`, `CategoryFormSchema`
- LucideIconPicker bound via `field.handleChange` (D-08 pattern)
- ColourPicker bound via `field.handleChange` (D-08 pattern)
- Fields laid out with `FieldGroup` + `Field` â€” no `space-y-*` classes
- Mutation error surfaced via `form.setErrorMap({ onSubmit: ... })` + `form.Subscribe`
- Submit button state via `form.Subscribe selector={(s) => s.isSubmitting}`

**Commit:** `207e088`

### Task 2: Split RuleDialog into chrome + RuleForm (with null categoryId)

- **RuleDialog.tsx** rewritten to ~18 lines: Dialog chrome only, imports and renders `<RuleForm />`, MATCH_TYPE_LABELS moved out
- **RuleForm.tsx** created: TanStack Form with `useAppForm`, `zodValidator`, `RuleFormSchema`
- `categoryId` default is `null` (D-06) â€” `__none__` sentinel completely eliminated
- Radix Select null bridge: `value={field.state.value ?? ""}` + `onValueChange={(v) => field.handleChange(v === "" ? null : v)}`
- "No category" as `SelectValue placeholder` only â€” no `SelectItem value=""` (Radix runtime error prevention)
- Cross-field regex validator on blur: `fieldApi.form.getFieldValue("matchType")` reads sibling field
- Server-side `INVALID_REGEX` error uses `form.setFieldMeta("pattern", ...)` to target the pattern field
- Fields laid out with `FieldGroup` + `Field` â€” no `space-y-*` classes

**Commit:** `405fe87`

  <action>
    Create apps/web/src/components/accounts/AccountForm.tsx. This file has NO default export —
    only the named export AccountForm. One component per file rule (D-12) means AccountFormInner
    is a private non-exported component in the SAME file (it is not a separate export, not used
    outside this file).

    The file implements three things:
    1. Loading gate: AccountForm checks isLoading from useGetAccountMembers; renders Spinner while loading
    2. AccountFormInner: receives account + existingMembers as props; calls useAppForm with fully-loaded defaultValues
    3. The form JSX: identical fields to the current AccountSheet form (same labels, same placeholder text, same layout classes)

    Critical implementation rules:
    - Import: `import type { AccountForm as AccountFormType } from "@ploutizo/validators"` — alias required because AccountForm is also the component name (D-03b)
    - Import: `import { AccountFormSchema } from "@ploutizo/validators"`
    - Import: `import { useAppForm } from "@ploutizo/ui/components/form"`
    - Import: `import { zodValidator } from "@tanstack/zod-form-adapter"`
    - useAppForm options: `validatorAdapter: zodValidator()` MUST be present — without it, Zod validation does not fire (Pitfall 1)
    - useAppForm options: `validators: { onSubmit: AccountFormSchema }` for form-level submit validation
    - defaultValues must use `satisfies AccountFormType` with a COMPLETE object (not Partial) — see Pitfall 7
    - ownership and memberIds are first-class form fields (D-04) — no useState for these
    - onSubmit: use `value` parameter, never closure-captured state (Pitfall 4)
    - Mutation error: use `form.setErrorMap({ onSubmit: "Couldn't save changes. Check your connection and try again." })`
    - Submit button: use `form.Subscribe selector={(s) => s.isSubmitting}` — not mutation.isPending (Research Pattern 4)
    - Use `form.AppField` not `form.Field` for all fields (Pitfall 5)

    defaultValues in AccountFormInner:
    ```tsx
    const loadedMemberIds = existingMembers.map((m) => m.memberId)
    defaultValues: {
      name: account?.name ?? "",
      type: account?.type ?? "chequing",
      institution: account?.institution ?? "",
      lastFour: account?.lastFour ?? "",
      eachPersonPaysOwn: account?.eachPersonPaysOwn ?? false,
      ownership: loadedMemberIds.length > 0 ? "shared" : "personal",
      memberIds: loadedMemberIds,
    } satisfies AccountFormType
    ```

    Form field list (all via form.AppField):
    1. ownership — ToggleGroup type="single"; `if (v) field.handleChange(v as "personal" | "shared")`
    2. name — Input; `validators={{ onChange: AccountFormSchema.shape.name }}`; error display on `field.state.meta.errors[0]`
    3. type — Select; value=`field.state.value`; onValueChange=`field.handleChange`
    4. institution — Input (optional field, no onChange validator)
    5. lastFour — Input with `.replace(/\D/g, "").slice(0, 4)` filter in onChange; maxLength={4}; className="font-mono"
    6. ownership === "shared" conditional: co-owners Checkbox list — use form.Subscribe to read ownership value, then render Checkbox per member using `form.AppField name="memberIds"`
    7. eachPersonPaysOwn — inside Collapsible (advancedOpen as local useState is fine — it's UI-only expand state, not a form field); Checkbox bound to `form.AppField name="eachPersonPaysOwn"` with `onCheckedChange={(checked) => field.handleChange(Boolean(checked))}`

    Footer (inside the form element, after the scrollable content div):
    ```tsx
    <div className="flex justify-end gap-2 border-t border-border px-6 py-4">
      <Button variant="outline" type="button" onClick={onClose}>Discard</Button>
      <form.Subscribe selector={(s) => s.isSubmitting}>
        {(isSubmitting) => (
          <Button type="submit" disabled={isSubmitting}>
            {isSubmitting && <Spinner className="mr-1" />}
            {isEditing ? "Save changes" : "Add account"}
          </Button>
        )}
      </form.Subscribe>
    </div>
    ```

    onSubmit payload — strip ownership (UI-only field), send memberIds conditionally:
    ```tsx
    onSubmit: ({ value }) => {
      const payload = {
        name: value.name.trim(),
        type: value.type,
        institution: value.institution?.trim() || undefined,
        lastFour: value.lastFour?.trim() || undefined,
        eachPersonPaysOwn: value.eachPersonPaysOwn,
        memberIds: value.ownership === "shared" ? value.memberIds : [],
      }
      const mutation = isEditing ? updateAccount : createAccount
      mutation.mutate(payload, {
        onSuccess: onClose,
        onError: () => form.setErrorMap({ onSubmit: "Couldn't save changes. Check your connection and try again." }),
      })
    }
    ```

    - Each field group: `className="space-y-1"`
    - Label: `className="text-xs font-medium"`
    - Error paragraph: `className="text-xs text-destructive"`
    - Form element: `className="flex flex-1 flex-col overflow-hidden"`

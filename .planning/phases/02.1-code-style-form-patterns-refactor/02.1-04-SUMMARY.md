# Phase 02.1 Plan 04: HouseholdSettings + MerchantRuleRow Refactor Summary

**One-liner:** HouseholdSettings split into page shell and TanStack Form component; MerchantRuleRow extracted from MerchantRulesSettings reducing map body from 30+ inline lines to one component call.

## What Was Built

### Task 1: Split HouseholdSettings into shell + HouseholdSettingsForm

- Rewrote `HouseholdSettings.tsx` to a pure page shell with no useState, no data-access hooks, no business logic
- Replaced `space-y-*` with `flex flex-col gap-*` throughout (SKILL.md: "No space-y-*. Use flex with gap-*")
- Created `HouseholdSettingsForm.tsx` using `useAppForm` with `zodValidator()` and `HouseholdSettingsFormSchema` from `@ploutizo/validators`
- Inline save pattern: `$` prefix span + number input + Save button in a `flex items-center gap-2` row
- Cents/dollars conversion: `data.settlementThreshold / 100` for display, `Math.round(dollars * 100)` on submit
- Form-level error via `form.setErrorMap({ onSubmit: "..." })` and `form.Subscribe` render
- Per-field error via `field.state.meta.errors` and `aria-invalid`

**Commit:** `db5b60b`

### Task 2: Extract MerchantRuleRow from MerchantRulesSettings

- Created `MerchantRuleRow.tsx` with full SortableItem layout: drag handle (GripVertical + SortableItemHandle), match type label, pattern text, optional renameTo arrow, Edit button, and DeleteRuleButton
- `DeleteRuleButton` is a private non-exported component in the same file — AlertDialog with confirmation (D-12: one exported component per file; private helpers permitted)
- `MerchantRulesSettings.tsx` map body reduced from 30+ inline lines to a single `<MerchantRuleRow ... />` call
- Removed from MerchantRulesSettings: AlertDialog and all sub-imports, GripVertical, SortableItem, SortableItemHandle imports

**Commit:** `ec5aa42`

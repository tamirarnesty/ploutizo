## Decisions Made

- `validatorAdapter: zodValidator()` removed: TanStack Form 1.28.6 removed this from TypeScript interface (deprecated per d.ts). Zod validation still achieved via `AccountFormSchema.safeParse` in the `onSubmit` validator function. Runtime behavior identical.
- `validators.onSubmit` uses a function wrapper `({ value }) => AccountFormSchema.safeParse(value)` rather than a Zod schema literal. This preserves string type compatibility for `form.setErrorMap({ onSubmit: "..." })`.

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added @ploutizo/validators dep and tsconfig path to apps/web**
- **Found during:** Task 2 (AccountForm.tsx typecheck)

- **Fix:** `pnpm add --filter web "@ploutizo/validators@workspace:*"` + `pnpm add --filter web @tanstack/zod-form-adapter`; added `"@ploutizo/validators": ["../../packages/validators/src/index.ts"]` to tsconfig.json paths
- **Files modified:** apps/web/package.json, apps/web/tsconfig.json, pnpm-lock.yaml
- **Verification:** TypeScript resolves @ploutizo/validators correctly after fix
- **Committed in:** `6c6b4f4` (Task 2 commit)

**2. [Rule 1 - Bug] Removed validatorAdapter and switched validators.onSubmit to function wrapper**
- **Found during:** Task 2 (AccountForm.tsx typecheck)
- **Issue:** TanStack Form 1.28.6 removed `validatorAdapter` from TypeScript interface (deprecated; d.ts says "adapter is no longer needed with zod 3.24.0"). Also, passing a Zod schema literal to `validators.onSubmit` is not assignable to `FormValidateOrFn<TFormData>` when the schema's StandardSchemaV1 input type doesn't exactly match the inferred TFormData.
- **Fix:** Removed `validatorAdapter: zodValidator()` (not in types). Changed `validators.onSubmit` from `AccountFormSchema` to `({ value }) => { const r = AccountFormSchema.safeParse(value); if (!r.success) return r.error.issues.map(i => i.message).join(', ') }`. Field-level validator `validators={{ onChange: AccountFormSchema.shape.name }}` works correctly without adapter.
- **Files modified:** apps/web/src/components/accounts/AccountForm.tsx
- **Verification:** TypeCheck passes with zero errors on AccountForm.tsx and AccountSheet.tsx
- **Committed in:** `6c6b4f4` (Task 2 commit)

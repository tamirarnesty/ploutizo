# Phase 02.2: add light/dark/system theme toggle - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a light/dark/system theme toggle to the app with `system` as the default. The toggle is a single cycling icon button in the sidebar bottom area (left of UserButton). Clerk's auth UI adapts automatically via the existing shadcn theme. No Settings page, no DB persistence — localStorage only via next-themes.

</domain>

<decisions>
## Implementation Decisions

### Theme Library
- **D-01:** Use `next-themes` — install in `packages/ui` and export `ThemeProvider` + `useTheme` re-exports from `@ploutizo/ui`. Consistent with where shadcn and Sidebar live.
- **D-02:** Default theme: `system`. next-themes `defaultTheme="system"` with `enableSystem`.
- **D-03:** `storageKey` can stay at next-themes default (`theme`). localStorage only — no DB sync in v1.

### Toggle UI
- **D-04:** Single cycling icon button placed at the **bottom of the sidebar, to the left of `<UserButton />`** on the same row.
- **D-05:** Cycle order: **System → Light → Dark → System** (starts at System since that's the default).
- **D-06:** Icon per mode: Monitor (System), Sun (Light), Moon (Dark) — all from lucide-react (already installed).
- **D-07:** Tooltip on hover showing the current mode name: "System", "Light", or "Dark". Use shadcn `<Tooltip>` from `@ploutizo/ui`.

### Clerk Appearance
- **D-08:** Keep `ClerkProvider appearance={{ theme: shadcn }}` as-is. The shadcn Clerk theme automatically adapts to light and dark mode when the `.dark` class is present on an ancestor element — no dynamic appearance prop switching needed.

### Tailwind Integration
- **D-09:** `packages/ui/src/styles/globals.css` already has `.dark { ... }` CSS variable overrides and `@custom-variant dark (&:is(.dark *))`. next-themes adds the `.dark` class to `<html>` — this wires directly into the existing setup with no CSS changes needed.
- **D-10:** `ThemeProvider` must be added to the `<html>` element (or its immediate child) so the `.dark` class is applied at the root. In TanStack Start's `__root.tsx`, wrap inside `RootDocument` around `<html>` or configure `attribute="class"` on next-themes `ThemeProvider`.

### Claude's Discretion
- Exact Tailwind classes for the toggle button (size, hover state, active ring, etc.)
- Whether to suppress hydration warnings (`suppressHydrationWarning`) on `<html>` — standard next-themes requirement

</decisions>

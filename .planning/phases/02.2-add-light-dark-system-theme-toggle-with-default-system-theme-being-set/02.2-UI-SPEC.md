## ThemeToggle Interaction Spec

### Button anatomy

```
<Tooltip>
  <TooltipProvider>
    <TooltipTrigger asChild>
      <Button variant="ghost" size="icon" className="size-8">
        {theme === "system" && <MonitorIcon />}
        {theme === "light"  && <SunIcon />}
        {theme === "dark"   && <MoonIcon />}
        <span className="sr-only">{tooltipLabel}</span>
      </Button>
    </TooltipTrigger>
    <TooltipContent>{tooltipLabel}</TooltipContent>
  </TooltipProvider>
</Tooltip>
```

### Button classes

| Class                | Reason                                           |
|----------------------|--------------------------------------------------|
| `variant="ghost"`    | No background at rest — sidebar surface shows through |
| `size="icon"`        | shadcn icon button preset — handles padding      |
| `className="size-8"` | 32px touch target, matches sidebar compact scale |

No manual `dark:` overrides. Hover uses `hover:bg-sidebar-accent` / `hover:text-sidebar-accent-foreground` which are provided by `variant="ghost"` inside the sidebar context automatically via the radix-nova sidebar token set.

### Cycle order (D-05)

Current theme → onClick → next theme:

| Current | Next   |
|---------|--------|
| system  | light  |
| light   | dark   |
| dark    | system |

Implementation: `const next = { system: "light", light: "dark", dark: "system" } as const`

### Icon mapping (D-06)

| Theme  | Icon component | Import                         |
|--------|----------------|--------------------------------|
| system | `MonitorIcon`  | `import { Monitor as MonitorIcon } from "lucide-react"` |
| light  | `SunIcon`      | `import { Sun as SunIcon } from "lucide-react"`         |
| dark   | `MoonIcon`     | `import { Moon as MoonIcon } from "lucide-react"`       |

Icons render without explicit size classes — shadcn `size="icon"` button handles icon scaling via CSS.

### Tooltip (D-07)

| Theme  | Tooltip text |
|--------|--------------|
| system | "System"     |
| light  | "Light"      |
| dark   | "Dark"       |

Tooltip also serves as the screen-reader label via `<span className="sr-only">`.

## ThemeProvider Integration Spec

**Location:** `apps/web/src/routes/__root.tsx` — inside `RootDocument`, wrapping the body children.

**Placement rule:** `ThemeProvider` must be an ancestor of `<html>` body content so `.dark` is applied at document root. In TanStack Start the `<html>` element is rendered inside `RootDocument`. Wrap the content passed to `<body>` with `ThemeProvider`.

**Required props:**

```tsx
<ThemeProvider
  attribute="class"
  defaultTheme="system"
  enableSystem
  disableTransitionOnChange
  storageKey="theme"
>
  {children}
</ThemeProvider>
```

| Prop                       | Value     | Reason                                                          |
|----------------------------|-----------|-----------------------------------------------------------------|
| `attribute="class"`        | required  | Adds `.dark` class — matches `@custom-variant dark` in globals.css |
| `defaultTheme="system"`    | required  | D-02 locked decision                                            |
| `enableSystem`             | required  | Enables OS preference detection                                 |
| `disableTransitionOnChange`| required  | Prevents flash of unstyled content on theme switch              |
| `storageKey="theme"`       | default   | next-themes default — no change needed (D-03)                   |

**`suppressHydrationWarning` on `<html>`:** Add `suppressHydrationWarning` to the `<html lang="en">` element in `RootDocument`. This is the standard next-themes requirement — next-themes modifies the class attribute server-vs-client and React will warn without this attribute. This is safe and expected.

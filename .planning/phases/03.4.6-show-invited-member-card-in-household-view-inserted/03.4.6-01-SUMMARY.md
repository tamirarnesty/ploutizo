## Implementation Notes

### Clerk Timestamp Confirmation

Clerk timestamps are 10-digit Unix seconds (e.g., `1678886400`). Confirmed by RESEARCH.md A3 assumption — verified correct by the test asserting `new Date(1678886400 * 1000).toISOString()` produces a 2023 date (not 1970). Multiply by 1000 before `new Date()`.

### Route Handler Signatures

```typescript
// GET /invitations
householdsRouter.get('/invitations', async (c) => {
  const orgId = c.get('orgId'); // JWT-scoped, never client-supplied
  const rows = await listInvitations(orgId);
  return c.json({ data: rows }); // PendingInvitation[]
});

// DELETE /invitations/:invitationId
householdsRouter.delete('/invitations/:invitationId', async (c) => {
  const orgId = c.get('orgId');
  const { userId: requestingUserId } = getAuth(c);
  const { invitationId } = c.req.param();
  const result = await revokeInvitation(orgId, invitationId, requestingUserId ?? '');
  return c.json({ data: result }); // { revoked: true }
});
```

### Response Shapes

- `GET /invitations` → `{ data: PendingInvitation[] }` where each item has `{ id, email, status, createdAt, expiresAt }` (camelCase ISO strings, no Clerk snake_case)
- `DELETE /invitations/:invitationId` → `{ data: { revoked: true } }`

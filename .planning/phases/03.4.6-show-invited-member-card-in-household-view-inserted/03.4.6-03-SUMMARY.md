## Post-Checkpoint Fixes Detail

**Timestamp bug (Fix 1):** Clerk REST API returns timestamps in Unix milliseconds, not seconds. The original code multiplied by 1000 (`new Date(inv.created_at * 1000)`) producing dates ~50 years in the future. Removed the multiplier.

**Double-flash (Fix 2):** `isLoading = membersQuery.isLoading || invitationsQuery.isLoading` caused a skeleton to flash every navigation when TanStack Query served stale cached data while revalidating. Changed to `!membersQuery.data && !invitationsQuery.data` â€” skeleton only shows when neither query has ever resolved.

**Right-align + hover-only delete (Fixes 3-5):** `<ItemActions className="ml-auto">` right-aligns the actions block. The delete button uses `triggerClassName="opacity-0 group-hover/item:opacity-100 focus-visible:opacity-100 data-[state=open]:opacity-100"` â€” Item's cva applies `group/item` so this correctly reveals the button on row hover only.

**Tooltip (Fixes 3-5):** Added `tooltip` and `triggerClassName` props to `ConfirmDialog`. Base UI's `TooltipTrigger` does not support `asChild`, so the dialog was refactored to controlled mode (`open`/`onOpenChange` on `AlertDialog`) with a plain `<button>` that `TooltipTrigger` wraps via its `render` prop.

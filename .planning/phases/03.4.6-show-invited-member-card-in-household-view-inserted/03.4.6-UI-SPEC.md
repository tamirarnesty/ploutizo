## Component Inventory

### New: `InvitedMemberRow`

Structural mirror of `MemberRow` — same `<li>` layout, same `px-4 py-3` padding, same `flex items-center justify-between` structure. Visually distinct via `opacity-60`.

```

  <!-- Left: avatar + email + "Invited" badge -->
  <div className="flex min-w-0 items-center gap-2">
    <UserAvatar name={email} className="shrink-0" />        {/* email → initials */}
    <div className="flex min-w-0 flex-col">
      <Text as="span" variant="body-sm" className="min-w-0 truncate font-semibold">
        {email}
      </Text>
      <div className="flex items-center gap-1">

          Invited {relativeTime(createdAt)}
        </Text>
        <Text as="span" variant="caption" className={expiryClass}>
          · {expiryLabel}
        </Text>
      </div>
    </div>

  <!-- Right: revoke button -->
  <div className="ml-4 flex shrink-0 items-center">
    <AlertDialog>
      <AlertDialogTrigger
        className="inline-flex size-9 items-center justify-center rounded-md text-muted-foreground hover:bg-destructive/10 hover:text-destructive focus-visible:ring-2 focus-visible:ring-ring focus-visible:outline-none"
        aria-label={`Revoke invite to ${email}`}
      >
        <Trash2 className="size-4" />
      </AlertDialogTrigger>
      ...
    </AlertDialog>
  </div>
</li>
```

Source: CONTEXT.md D-08, D-09, D-10, D-11, D-12. Icon: `Trash2` from Lucide (same as `MemberRow`).

### Expiry label logic

| Condition | Text | Color class |
|-----------|------|-------------|
| `status === 'expired'` | "Expired" | `text-destructive` |
| `status === 'pending'` AND days remaining ≤ 2 | "Expires in {N} day(s)" | `text-warning-foreground` |
| `status === 'pending'` AND days remaining > 2 | "Expires in {N} days" | `text-muted-foreground` |

Days remaining is computed from `expiresAt` for display label text only — Clerk `status` field is authoritative for the "Expired" state. Source: CONTEXT.md D-11.

### Reused: `UserAvatar`

Pass `name={email}` — `getInitials` will extract initials from the email string. No `imageUrl` for pending members. Source: CONTEXT.md D-09; code: `UserAvatar.tsx`.

### Reused: `Badge variant="secondary"`

"Invited" badge — same variant as "You" badge in `MemberRow`. Source: CONTEXT.md code_context.

### Reused: `AlertDialog` (revoke)

Same pattern as `MemberRow` remove flow. Source: CONTEXT.md D-12.

## States & Interactions

| State | Trigger | Rendering |
|-------|---------|-----------|
| Loading | `membersQuery.isLoading OR invitationsQuery.isLoading` | 3 skeleton rows (single combined block, no progressive loading) |
| Populated — active + invited | Both queries settled | Active `MemberRow` items first, then `InvitedMemberRow` items |
| Populated — invited only (no active) | Members empty, invitations present | `InvitedMemberRow` items render; no empty state text |
| Empty | Both queries empty | "This household has no other members." (`text-muted-foreground`, `body-sm`) |
| Revoke pending (mutation in-flight) | User clicks Revoke in dialog | Button shows loading indicator (existing pattern — match `useRemoveMember` UX) |
| Warning expiry | `status === 'pending'` AND ≤ 2 days to `expiresAt` | Expiry label: `text-warning-foreground` |
| Expired | `status === 'expired'` | Expiry label: "Expired" in `text-destructive` |

Source: CONTEXT.md D-11, D-13, D-14.

## Interaction Patterns

### Revoke flow
1. User clicks trash icon on `InvitedMemberRow` — `AlertDialog` opens.
2. User clicks "Revoke" — `useRevokeInvitation().mutate(invitationId)` fires.
3. On settle — `['org-invitations']` invalidated; card disappears from list.
4. Dialog closes on both Revoke and "Keep invite".

No toast required (matches existing `useRemoveMember` pattern — check `MemberRow` UX for exact feedback).

### Invite → card appears
After `useInviteMember` settles, it now invalidates `['org-invitations']` in addition to `['org-members']`. The new invitation card appears immediately in the list without a page reload. Source: CONTEXT.md D-07.

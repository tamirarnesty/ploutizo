## Spacing Scale

Declared values (multiples of 4 only). Maps to Tailwind's default spacing scale.

| Token | Value | Usage                                                          |
| ----- | ----- | -------------------------------------------------------------- |
| xs    | 4px   | Icon-to-label gaps, badge inner padding, inline chip spacing   |
| sm    | 8px   | Compact element spacing, list item gap, icon button padding    |
| md    | 16px  | Default element spacing, form field gap, card body padding     |
| lg    | 24px  | Section padding, card header padding, sheet body padding       |
| xl    | 32px  | Layout content inset (main content left padding), dialog body  |
| 2xl   | 48px  | Major section breaks between page regions                      |
| 3xl   | 64px  | Page-level top/bottom padding on standalone pages (onboarding) |

Exceptions:

- Sidebar width: **240px** fixed (D-05 from CONTEXT.md — fixed, no collapse)
- Touch targets for icon-only buttons (hamburger, close, drag handle): minimum **44px** height per WCAG 2.5.5
- Clerk `<OrganizationSwitcher />` and `<UserButton />` use Clerk's internal spacing — do not override
- **16px (`py-4`)**: Sheet header and sheet body top/bottom padding (`px-6 py-4`) — aligns with `md` token for sheet chrome
- **8px (`px-2` / `py-2` / `gap-2`)**: Nav item padding (`px-2 py-2`) and sheet footer button gap (`gap-2`) — `sm` token for compact nav and footer layouts (use `px-4` / `gap-4` if more padding is preferred)
- **32px (`h-8`) nav item height**: Compact sidebar nav row height; use **40px (`h-10`)** for a more standard row height

## Color

All values reference the CSS custom properties defined in `globals.css`. Use the token names, not raw hex/OKLCH values in component code.

### Primary Token Reference

| Token           | Light value (OKLCH)                           | Dark value (OKLCH)                        | Tailwind class                        |
| --------------- | --------------------------------------------- | ----------------------------------------- | ------------------------------------- |
| `--background`  | `oklch(1 0 0)` — white                        | `oklch(0.147 0.004 49.25)` — near-black   | `bg-background`                       |
| `--card`        | `oklch(1 0 0)` — white                        | `oklch(0.216 0.006 56.043)` — dark card   | `bg-card`                             |
| `--sidebar`     | `oklch(0.985 0.001 106.423)` — warm off-white | `oklch(0.216 0.006 56.043)`               | `bg-sidebar`                          |
| `--primary`     | `oklch(0.5 0.134 242.749)` — blue             | `oklch(0.443 0.11 240.79)` — darker blue  | `bg-primary` / `text-primary`         |
| `--destructive` | `oklch(0.577 0.245 27.325)` — red             | `oklch(0.704 0.191 22.216)` — lighter red | `bg-destructive` / `text-destructive` |
| `--muted`       | `oklch(0.97 0.001 106.424)` — warm light gray | `oklch(0.268 0.007 34.298)` — dark muted  | `bg-muted` / `text-muted-foreground`  |

### 60/30/10 Color Contract

| Role            | Token                                    | Usage                                                                                                                                              |
| --------------- | ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `bg-background`                          | Main page background, full-width page areas                                                                                                        |
| Secondary (30%) | `bg-sidebar` / `bg-card` / `bg-muted`    | Sidebar surface, cards, DataGrid rows, form field backgrounds, sheet panels, popover backgrounds                                                   |
| Accent (10%)    | `bg-primary` / `text-primary`            | Active sidebar nav indicator, primary CTA buttons ("Add Account", "Save changes", "Archive"), focus rings via `ring-ring`, drag handle hover state |
| Destructive     | `text-destructive` / `bg-destructive/10` | Archive confirmation dialogs, delete action buttons, inline regex validation error text                                                            |

Accent reserved for (exhaustive list):

1. Primary CTA buttons (the single most important action per page)
2. Active/selected state indicator on sidebar nav items (`bg-sidebar-primary` / `text-sidebar-primary-foreground`)
3. Focus rings on interactive elements (`ring-ring/50`)
4. Drag handle active state in ReUI Sortable lists
5. Icon picker selected icon highlight (category icon selection popover)

## States & Interactions

### Loading States

| Context                                                | Pattern                                                                      |
| ------------------------------------------------------ | ---------------------------------------------------------------------------- |
| Page data fetch (accounts list, categories list)       | Skeleton rows — 5 rows at same height as real rows, `bg-muted animate-pulse` |
| Sheet form open (fetching member list for Shared mode) | Spinner (`size-4 animate-spin`) inline next to "Co-owners" label             |
| Mutation in-flight (save button clicked)               | Button shows spinner + disabled state; label unchanged                       |
| Inline create (tag POST in-flight)                     | Combobox input disabled, spinner appears where "Create 'X'" item was         |

### Error States

| Context                             | Pattern                                                                               |
| ----------------------------------- | ------------------------------------------------------------------------------------- |
| Form validation (Zod issues)        | Field-level `<FormMessage>` below each invalid field — `text-destructive text-xs`     |
| Merchant rule regex (on blur)       | Inline `text-destructive text-xs` below pattern input — "Invalid regular expression." |
| CRUD mutation failure (API 4xx/5xx) | Toast notification: `variant: destructive`, text from Copywriting Contract above      |
| Empty query result                  | Empty state component (heading + body + optional CTA) per Copywriting Contract        |

### Interaction Contracts

| Interaction                          | Behavior                                                                                                                                   |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Sidebar nav click                    | TanStack Router `<Link>` navigation; active class applied to matching route                                                                |
| Add account sheet                    | Opens via shadcn `<Sheet>` controlled by local `open` state; closes on Save or Discard or Escape                                           |
| Ownership toggle (Personal → Shared) | Hides/shows Co-owners multi-select with `AnimatePresence` or conditional render; does not reset form                                       |
| Advanced collapsible open            | `<Collapsible>` with chevron icon rotation (180deg, `transition-transform duration-200`)                                                   |
| Category drag-to-reorder             | ReUI Sortable `onValueChange` fires with new order; optimistic update applied; PATCH to API on drop                                        |
| Merchant rule drag-to-reorder        | Same pattern as category reorder                                                                                                           |
| Icon picker search                   | Filters curated icon array on `input` event (no API call); renders matching icons live                                                     |
| Tag combobox create                  | "Create 'X'" item appears when `inputValue` has no match; fires POST mutation; closes popover on success; selected tag added to form state |
| Archive confirmation                 | shadcn `<AlertDialog>` controlled open; blocks background interaction; "Archive" confirm fires mutation; closes on success                 |
| Delete rule confirmation             | Same `<AlertDialog>` pattern as archive                                                                                                    |
| Regex field blur                     | `try { new RegExp(value) } catch` — if invalid, set `isRegexError: true` in local state; clear on next valid input                         |
| Mobile hamburger                     | Opens sidebar as `<Sheet side="left">`; `aria-label="Open navigation"` on trigger button; closes on nav click or outside click             |

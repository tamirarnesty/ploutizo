## Canonical Pattern

**Source:** `apps/web/src/components/settings/CategoryForm.tsx` lines 90–94

All 9 files converge on a single fix. The canonical FieldError usage pattern is:

```tsx
{field.state.meta.errors.length > 0 ? (
  <FieldError
    errors={field.state.meta.errors as { message?: string }[]}
  />
) : null}
```

**Import (field import block, lines 7–11 of CategoryForm):**
```tsx
import {
  Field,
  FieldError,
  FieldGroup,
  FieldLabel,
} from '@ploutizo/ui/components/field';
```

**FieldError component signature** (`packages/ui/src/components/field.tsx` lines 174–180):
```tsx
function FieldError({
  className,
  children,
  errors,
  ...props
}: React.ComponentProps<'div'> & {
  errors?: ({ message?: string } | undefined)[];
})
```
Returns `null` when `errors` is empty and no `children` are provided — unconditional render is safe, but the `length > 0` guard matches the canonical form and should be kept.

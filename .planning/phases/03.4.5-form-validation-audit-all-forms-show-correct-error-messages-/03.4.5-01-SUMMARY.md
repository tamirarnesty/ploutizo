## Outcome

All forms now render human-readable validation error messages instead of `[Object object]`.

**Root cause:** TanStack Form's `ValidationError` is an object with a `.message` property. Rendering via `String(error)`, `error.toString()`, or `{error}` (bare JSX) produces `[Object object]`. `FieldError` with the `errors` prop correctly extracts `.message`, deduplicates, and renders with `role="alert"` and `text-destructive` styling.

**Sites fixed:**
- `AccountForm.tsx` — 1 site (children → errors prop)
- `HouseholdSettingsForm.tsx` — 1 site (`<Text variant="error">` → FieldError) + FieldError import added
- `InviteMemberForm.tsx` — 1 site (`<Text variant="error">` → FieldError) + FieldError import added, unused Text import removed
- `RuleForm.tsx` — 1 site (`?.toString()` children → errors prop)
- `TransactionForm.tsx` — 5 sites (children → errors prop)
- `TransferFields.tsx` — 1 site (children → errors prop)
- `ExpenseFields.tsx` — 1 site (children → errors prop)
- `SettlementFields.tsx` — 2 sites (children → errors prop)
- `TransactionTypeFields.tsx` — 1 new site added in `IncomeTypeField` + FieldError added to field import

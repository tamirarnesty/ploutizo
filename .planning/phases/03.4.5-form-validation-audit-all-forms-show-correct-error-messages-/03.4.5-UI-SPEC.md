## Error State Rendering Contract

This section is the primary deliverable of this phase. It defines exactly how every `field.state.meta.errors` site must render.

### Canonical Component

```tsx
{field.state.meta.errors.length > 0 ? (
  <FieldError
    errors={field.state.meta.errors as { message?: string }[]}
  />
) : null}
```

Source: `apps/web/src/components/settings/CategoryForm.tsx` (reference implementation).

### FieldError Visual Behavior

| State | Rendered Output |
|-------|----------------|
| No errors (empty array) | Returns `null` — no DOM node |
| Single error | `<div role="alert" data-slot="field-error" class="text-sm font-normal text-destructive">message string</div>` |
| Multiple errors (deduplicated) | `<div role="alert" ...><ul class="ml-4 flex list-disc flex-col gap-1"><li>…</li></ul></div>` |
| Duplicate messages | Deduplication via `Map` by `.message` — only unique messages rendered |

Source: `packages/ui/src/components/field.tsx` lines 174–223 (verified 2026-05-01).

### Accessibility

- `role="alert"` on the wrapper `<div>` — announced to screen readers on appearance
- `data-slot="field-error"` attribute — for test selection and CSS targeting
- No ARIA additions needed beyond what FieldError already provides

### Anti-Patterns (all produce `[Object object]` — forbidden)

| Pattern | Why Forbidden |
|---------|--------------|
| `<FieldError>{String(field.state.meta.errors[0])}</FieldError>` | Coerces ValidationError object to string |
| `<FieldError>{field.state.meta.errors[0]?.toString()}</FieldError>` | Same object coercion |
| `{field.state.meta.errors[0]}` bare in JSX | React renders object reference |
| `<Text variant="error">{field.state.meta.errors[0]}</Text>` | Non-canonical; same coercion |
| `<FieldError>` with children (no `errors` prop) | Children path bypasses `.message` extraction |

## Copywriting Contract

This phase introduces no new copy. All field labels, placeholders, and CTAs are unchanged.

| Element | Copy |
|---------|------|
| Primary CTA | N/A — no new submit buttons in this phase |
| Empty state | N/A — no new list views in this phase |

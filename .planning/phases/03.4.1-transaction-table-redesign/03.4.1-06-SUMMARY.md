## Accomplishments

### Task 1: AssigneeSection.tsx (new)

- One `Toggle` per org member in `flex flex-wrap gap-2` row
- Toggle pressed state: `data-[state=on]:bg-primary/10 data-[state=on]:text-primary data-[state=on]:border-primary/30` overridden at usage site (CLAUDE.md: never modify toggle.tsx)
- On press: adds member to split, recalculates LRM even split via `lrmSplit(amountCents, newMemberIds)`
- On un-press: removes member, recalculates LRM across remaining pressed members
- D-17: single org member auto-pressed in create mode; 2+ all un-pressed
- D-18: edit mode initializes pressed IDs from `transaction.assignees`; non-even distribution opens Collapsible
- "Customize split" Collapsible with ChevronDown/Up rotation; only renders when at least one member pressed
- `CollapsibleContent` renders `SplitSection` with filtered `pressedMembers` only
- Avatar with initials fallback; `AvatarImage` shown when `member.imageUrl` is set

### Task 2: TransactionForm.tsx (modified)

- **Merchant field removed** — entire `form.AppField name="merchant"` block deleted
- **Locked description** (D-11, D-12): `form.Subscribe` on `type + accountId + counterpartAccountId + refundOf`; computes locked templates from account lookup; shows `InputGroup` with `Lock` icon + readonly `InputGroupInput` + `SquarePen` unlock button with `Tooltip`
- **Lock templates implemented**: transfer (`Transfer from X to Y`), settlement (`Settlement — paid from X` or `Settlement`), contribution (`X contribution`), linked refund (static placeholder)
- **UnlockResetter**: invisible helper component using `useEffect` + `useRef` to reset unlock state on type transitions (avoids TransactionTypeFields modifications since Plan 05 is parallel)
- **Notes field** (D-21): `Textarea` rows=2, resize-none, `autoComplete="off"`, placeholder `Add a note…` (U+2026)
- **Notes + Tags layout**: `flex gap-4` row, both `flex-1` for equal width
- **AssigneeSection wired**: replaces `SplitSection` inside existing `form.Subscribe` + `form.AppField name="assignees"` wrappers; passes `transaction` prop for edit-mode init

## Deviations from Plan

### Auto-adapted: value/onChange/amountCents interface on AssigneeSection

**Found during:** Task 1 design
**Issue:** Plan task action specified `{ form, orgMembers, transaction }` interface on AssigneeSection, but PATTERNS.md and the Task 2 swap instructions specify `{ value, onChange, amountCents, orgMembers, transaction }` — the SplitSection-style composition. The latter follows `state-decouple-implementation` and matches the existing wrappers in TransactionForm.
**Fix:** Used `value/onChange/amountCents` interface. TransactionForm mounts AssigneeSection inside the existing `form.Subscribe` + `form.AppField` wrappers (identical to how SplitSection was mounted).
**Files modified:** Both files adjusted consistently.

### Auto-adapted: Type-change reset via UnlockResetter (no TransactionTypeFields modification)

**Found during:** Task 2 — `onTypeChange` prop on TransactionTypeFields
**Issue:** Plan specified resetting `isDescriptionUnlocked` via `onTypeChange` prop on `TransactionTypeFields`. But Plan 05 (which adds that prop) runs in Wave 4 parallel to Plan 06 — no guarantee it exists. TransactionTypeFields.tsx also still references removed fields (`incomeSource`, `toAccountId`, `settledAccountId`, `investmentType`) from Plan 04's cleanup — Plan 05's work.
**Fix:** Implemented reset via `UnlockResetter` helper component rendered inside `form.Subscribe` on `type` — no changes to TransactionTypeFields.tsx.
**Files modified:** TransactionForm.tsx only.

### Known limitation: Linked refund locked description

**Issue:** Plan specified `Refund of {originalDescription}` using `useGetTransaction(refundOf)`. Loading that query inside a `form.Subscribe` render callback would require additional query infrastructure not in scope.
**Current behavior:** Shows static `Refund of [original transaction]` placeholder. The lock is cosmetic (T-03.4.1-L1: accepted).
**Status:** Tracked as known stub — original description lookup can be added in a follow-up plan.

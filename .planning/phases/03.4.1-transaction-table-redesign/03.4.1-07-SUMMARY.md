## What Was Built

### Amount Column (D-22)

Replaced existing amount renderer with signed, color-coded output:

- `expense` → `−$X.XX` (U+2212 MINUS SIGN) in `text-destructive`
- `income` / `refund` → `+$X.XX` in `text-emerald-600 dark:text-emerald-400`
- `transfer` / `settlement` / `contribution` → `$X.XX` (no sign) in `text-muted-foreground`

Previous code incorrectly applied `text-destructive` to settlement and `text-emerald-*` to contribution.

### Account Column (D-23)

Replaced existing renderer to show `{accountName} → {counterpartAccountName}` (U+2192) when `counterpartAccountName` is present. Wrapped in `<div className="min-w-0">` with `Text className="truncate min-w-0"` for proper truncation. Column size increased from 160 to 220, minWidth meta updated to `min-w-[180px]`.

### Refund Sub-line (D-24)

Rendered inside the description cell as a `<button type="button">` below the description text. Visible only when `type === 'refund' && refundOfId !== null`. Content: `↩ {MMM D, YYYY} · {formatCurrency(refundOfAmountCents)}` using U+21A9, U+00B7. Clicking calls `onOpenOriginal(refundOfId)` which resolves the original transaction from the loaded page and opens it in `TransactionSheet` edit mode.

### Type Badge Opacity (UI-SPEC)

Added `isInternalType` predicate. Applied `opacity-60` via `cn()` to Badge for transfer, settlement, contribution.

### Internal Filter Shortcut (D-25)

Added `{ value: 'transfer,settlement,contribution', label: 'Internal' }` to the type filter options in `TransactionFilterFields.tsx`. The Filters component resolves the label "Internal" for the chip display. The comma-separated value passes through `transactionSearch.ts` unchanged (field is `z.string().optional()`) and is forwarded as-is to the API `type` query param.

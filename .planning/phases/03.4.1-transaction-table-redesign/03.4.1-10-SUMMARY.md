# Plan 10 Summary: 4 UAT Gap Fixes

## What was built

4 targeted fixes closing UAT gaps found after plan 09 execution:

1. **Contribution picker trigger** ‚Äî `ContributionFields.tsx` trigger now shows `a.name` only. The `INVESTMENT ‚Äî Name` prefix is preserved in dropdown options but stripped from the selected-value display.

2. **Stale query cache after edit** ‚Äî `useUpdateTransaction` now calls `qc.setQueryData(['transaction', id], updatedRow)` in `onSuccess`, so reopening the edit sheet immediately reflects the saved values without a page refresh.

3. **Same-account transfer guard** ‚Äî `TransferFields.tsx` extracted an inner `TransferDestinationField` component so `useMemo` and `useEffect` can run at a proper React component boundary. The destination list excludes the currently-selected source account; the effect clears a stale destination selection when the source changes mid-session.

4. **Date filter overhaul** ‚Äî `TransactionFilterFields.tsx` rewritten with:
   - Explicit 6-operator list (`is / is not / before / after / between / not between`) on the `dateRange` field ‚Äî overrides ReUI's duplicate-`is` default that caused the React key warning.
   - Operator-aware calendar mode: single-date operators show one month; range operators show two.
   - Trigger label: single-date ops show `Apr 20, 2026`; range ops show `Apr 1 ‚Äì Apr 20`.
   - Value migration on operator family switch (single‚Üîrange) via `valuesRef` snapshot to avoid stale closure capture.
   - `filtersToSearch` + `searchToFilters` in `Transactions.tsx` handle all 6 operators across page refresh.
   - `buildConditions` in API extended with `eq`/`ne`/`not(and(...))` for `is`/`is_not`/`not_between`.

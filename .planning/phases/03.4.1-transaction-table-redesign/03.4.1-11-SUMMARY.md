# Plan 11 Summary: Assignee Chips + Split Section UX Fixes

## What was built

2 UAT gaps closed (D-UAT-01 and D-UAT-02):

1. **D-UAT-01 — First name only in toggle chips** (`AssigneeSection.tsx`): `member.displayName.split(' ')[0]` replaces the full display name. Removed `min-w-[120px]` so chips size to their content.

2. **D-UAT-02 — Split section UX overhaul** (`SplitSection.tsx` + `AssigneeRow.tsx`):
   - **Removed "Add assignee" Select** and all dead code (`handleAddAssignee`, `availableMembers`). Assignees are now driven entirely by the toggle chips in `AssigneeSection`.
   - **Auto-remainder**: `handleRowChange` now distributes `(total - edited)` evenly across other rows after every keystroke — both `% mode` (distributes remaining percentage) and `$ mode` (distributes remaining cents with penny-rounding). `Math.max(0, ...)` guard prevents negative remainders when user types >100%.
   - **Tightened error condition**: `Math.abs(totalPct - 100) > 0.5` replaces the integer-rounding check. Error only surfaces when auto-remainder genuinely fails (degenerate state).
   - **InputGroup decorators** (`AssigneeRow.tsx`): Replaced plain `<Input>` with `<InputGroup className="w-24 shrink-0">` + `<InputGroupAddon align="inline-end">%</InputGroupAddon>` (percent mode) and `<InputGroupAddon align="inline-start">$</InputGroupAddon>` (dollar mode).
   - **Name truncation** (D-UAT-02 item 3): `min-w-0 flex-1 truncate` already present on the name span — confirmed and kept.

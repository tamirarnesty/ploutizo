# Plan 12 Summary: UAT Gap Closure (7 issues)

## What was built

7 UAT gaps closed after human testing revealed them post-Plan-11:

### Issue 7 — Type-switch crash (BLOCKER)
- **`useTransactionForm.ts`**: `buildDefaultValues` now uses `(transaction.tags ?? []).map(...)` and `(transaction.assignees ?? []).map(...)` — prevents crash when cache entry has undefined arrays.
- **`useUpdateTransaction.ts`**: `setQueryData` merges the PATCH response with the existing cache entry, preserving `tags`/`assignees` arrays that the scalar-only PATCH endpoint does not return.

### Issues 2+3 — Split inputs: free typing + decorator position
- **`AssigneeRow.tsx`**: Switched both inputs from `type="number"` to `type="text" inputMode="decimal"`. Added `isFocusedRef` gate so incoming prop changes don't clobber mid-keystroke display state. Both `%` and `$` decorators moved to `inline-start` (left side).

### Issue 4 — Hide "Customize split" for single assignee
- **`AssigneeSection.tsx`**: Guard changed from `pressedMemberIds.length > 0` to `pressedMemberIds.length > 1`.

### Issue 1 — Require ≥1 assignee
- **`TransactionForm.tsx`**: Added `validators={{ onChange }}` to the `assignees` AppField with "At least one assignee is required." message. Wrapped in `Field`/`FieldError`.

### Issues 5+6 — Refund: real description + pre-fill assignees
- **`TransactionForm.tsx`**: Added `RefundDataLoader` helper component that calls `useGetTransaction(refundOfId)` and fires callbacks when the original transaction loads. Renders via `form.Subscribe` on the `refundOf` field.
  - Locked description now shows `"Refund of [original description]"` instead of a static placeholder.
  - `refundOriginalAssigneeIds` passed to `AssigneeSection` in create mode.
- **`AssigneeSection.tsx`**: Added `refundAssigneeIds` prop + `useEffect` that syncs `pressedMemberIds` when the prop arrives (create mode only).

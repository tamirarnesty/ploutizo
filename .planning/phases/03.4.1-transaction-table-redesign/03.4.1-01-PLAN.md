  <name>Task 2: [BLOCKING] Generate, edit, and apply migration</name>

  <what-built>Drizzle schema updated in Task 1. Now the migration must be generated, manually corrected, and applied.</what-built>

  <how-to-verify>
**CRITICAL: Drizzle Kit generates DROP COLUMN before ADD COLUMN. You MUST manually edit the SQL before applying.**

Step 1 — Generate the migration:
```bash
pnpm --filter @ploutizo/db migrate:generate
```
This creates a new `.sql` file in `packages/db/drizzle/`.

Step 2 — Open the generated `.sql` file and REORDER + ADD backfill DML so the SQL runs in this exact order:

```sql
-- 1. ADD new nullable columns first (safe — no data loss)
ALTER TABLE "transactions" ADD COLUMN "counterpart_account_id" uuid REFERENCES "accounts"("id") ON DELETE SET NULL;
ALTER TABLE "transactions" ADD COLUMN "raw_description" text;
ALTER TABLE "transactions" ADD COLUMN "notes" text;

-- 2. Backfill counterpart_account_id from former FK columns
UPDATE "transactions" SET "counterpart_account_id" = "to_account_id" WHERE type = 'transfer' AND "to_account_id" IS NOT NULL;
UPDATE "transactions" SET "counterpart_account_id" = "settled_account_id" WHERE type = 'settlement' AND "settled_account_id" IS NOT NULL;

-- 3. Backfill description: use merchant value, fall back to '<no description>'
UPDATE "transactions" SET "description" = COALESCE("merchant", '<no description>') WHERE "description" IS NULL;

-- 4. Backfill raw_description from merchant for imported rows
UPDATE "transactions" SET "raw_description" = "merchant" WHERE "import_batch_id" IS NOT NULL AND "merchant" IS NOT NULL;

-- 5. Set description NOT NULL (safe now that backfill is complete)
ALTER TABLE "transactions" ALTER COLUMN "description" SET NOT NULL;

-- 6. DROP the old columns (safe — data preserved in new columns above)
ALTER TABLE "transactions" DROP COLUMN "merchant";
ALTER TABLE "transactions" DROP COLUMN "income_source";
ALTER TABLE "transactions" DROP COLUMN "to_account_id";
ALTER TABLE "transactions" DROP COLUMN "settled_account_id";
ALTER TABLE "transactions" DROP COLUMN "investment_type";
```

**The generated SQL will NOT have this order.** Drizzle Kit typically puts DROP before ADD. Move the SQL statements to match the order above. Add the UPDATE backfill statements (Drizzle Kit does not generate DML).

Step 3 — Apply the corrected migration:
```bash
pnpm --filter @ploutizo/db migrate:apply
```

Step 4 — Verify:

### `packages/db/drizzle/0004_*.sql` (migration, batch)

**Analog:** `packages/db/drizzle/0001_remarkable_callisto.sql` — `ALTER TABLE ... ADD COLUMN` + FK constraint pattern

**Migration file pattern** (0001 lines 20–23):
```sql
ALTER TABLE "orgs" ADD COLUMN "name" text;--> statement-breakpoint
ALTER TABLE "account_members" ADD CONSTRAINT "account_members_account_id_accounts_id_fk" FOREIGN KEY ("account_id") REFERENCES "public"."accounts"("id") ON DELETE cascade ON UPDATE no action;--> statement-breakpoint
```

**Required ordering for this migration** (from RESEARCH.md Pattern 1 — exact SQL, manually verified):
```sql
-- Step 1: ADD new nullable columns (Drizzle generates this portion)
ALTER TABLE "transactions" ADD COLUMN "counterpart_account_id" uuid REFERENCES "public"."accounts"("id") ON DELETE restrict ON UPDATE no action;--> statement-breakpoint
ALTER TABLE "transactions" ADD COLUMN "raw_description" text;--> statement-breakpoint
ALTER TABLE "transactions" ADD COLUMN "notes" text;--> statement-breakpoint

-- Step 2: MANUALLY INSERT — backfill counterpart_account_id from old FK columns
UPDATE transactions SET counterpart_account_id = to_account_id
  WHERE type = 'transfer' AND to_account_id IS NOT NULL;
UPDATE transactions SET counterpart_account_id = settled_account_id
  WHERE type = 'settlement' AND settled_account_id IS NOT NULL;

-- Step 3: MANUALLY INSERT — backfill raw_description from merchant for imported rows
UPDATE transactions SET raw_description = merchant
  WHERE import_batch_id IS NOT NULL AND merchant IS NOT NULL;

-- Step 4: MANUALLY INSERT — backfill description NOT NULL before constraint
UPDATE transactions SET description = COALESCE(merchant, 'Imported transaction')
  WHERE description IS NULL;

-- Step 5: MANUALLY INSERT — add NOT NULL constraint (now safe)
ALTER TABLE transactions ALTER COLUMN description SET NOT NULL;

-- Step 6: DROP old columns (Drizzle generates — move to bottom after backfills)
ALTER TABLE "transactions" DROP COLUMN "merchant";--> statement-breakpoint
ALTER TABLE "transactions" DROP COLUMN "income_source";--> statement-breakpoint
ALTER TABLE "transactions" DROP COLUMN "to_account_id";--> statement-breakpoint
ALTER TABLE "transactions" DROP COLUMN "settled_account_id";--> statement-breakpoint
ALTER TABLE "transactions" DROP COLUMN "investment_type";--> statement-breakpoint
```

**Critical:** Run `pnpm --filter @ploutizo/db migrate:generate` first, then edit the `.sql` output to match this ordering before `migrate:apply`. Drizzle places DROPs before ADDs — that order must be reversed manually.

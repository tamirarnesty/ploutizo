Note on "rejects" for strippped fields: Zod strips unknown keys by default (passthrough is not the default). Use `.strict()` in tests to verify the schema does not DECLARE merchant/toAccountId/settledAccountId fields. Alternatively, check that accessing `.shape.merchant` on the schema's type returns undefined (not a Zod type object).

Practical test approach: Parse a valid payload omitting the deprecated field and verify `success: true`. Then verify that the SCHEMA SHAPE does not contain the old key by checking `'merchant' in schema.shape === false` (for object schemas) or by running the type-level check via `z.ZodError` on `.strict()` parse.

## Color

Inherits all tokens from 03.4. The following are phase-specific additions and **breaking changes** from existing `TransactionColumns.tsx` color logic.

### Amount Column — Breaking Change

Current code has: `settlement=text-destructive`, `contribution=text-emerald-*`. This phase changes both.

| Transaction Type | Sign | Color Token | Tailwind Class |
|-----------------|------|-------------|----------------|
| expense | `-$X.XX` (negative) | `--destructive` | `text-destructive` |
| income | `+$X.XX` (positive) | `--success` (emerald-500/400) | `text-emerald-600 dark:text-emerald-400` |
| refund | `+$X.XX` (positive) | `--success` (emerald-500/400) | `text-emerald-600 dark:text-emerald-400` |
| transfer | `$X.XX` (no sign) | `--muted-foreground` | `text-muted-foreground` |
| settlement | `$X.XX` (no sign) | `--muted-foreground` | `text-muted-foreground` |
| contribution | `$X.XX` (no sign) | `--muted-foreground` | `text-muted-foreground` |

Formatting rule: sign prefix is prepended to the `formatCurrency(amountCents)` output string (`"−$X.XX"` / `"+$X.XX"`) — use U+2212 MINUS SIGN for expense negative, `+` ASCII for income/refund positive.

### Locked Description Field

| State | Visual |
|-------|--------|
| Locked | `bg-muted text-muted-foreground` Input, `cursor-default`, lock icon (`Lock`, 14px, `text-muted-foreground`) prepended inside field; "Edit" inline text button right side |
| Unlocked | Normal `Input` — same as all other description fields |

Lock/unlock toggle is UI state only — no schema flag. "Edit" button uses `variant="ghost"` `size="sm"` `className="h-auto py-0 px-1 text-xs"`.

### Internal Type Badge Opacity

Internal types (transfer, settlement, contribution) render their type Badge at `opacity-60` to visually de-emphasize them relative to financial types (expense, income, refund).

```
className={cn(typeBadgeClassName[type], isInternalType(type) && 'opacity-60')}
```

Where `isInternalType = (t: string) => ['transfer', 'settlement', 'contribution'].includes(t)`.

## Interaction Contracts — Phase 03.4.1 Deltas

### Table — Amount Column

Replace existing amount column cell renderer in `TransactionColumns.tsx`.

- Format: `formatCurrency(amountCents)` with sign prefix per the color table above
- Alignment: `text-right` (unchanged)
- No column width change
- Expense shows U+2212 minus (−), not hyphen-minus (-)
- Income and refund show `+` prefix

### Table — Account Column

Replace existing account column cell renderer.

- Primary-only (no counterpart): render `accountName` as `Text variant="body-sm" className="text-muted-foreground"`
- With counterpart (`counterpartAccountName` present): render `{accountName} → {counterpartAccountName}` as a single `Text` node
- Glyph: `→` (U+2192 RIGHTWARDS ARROW) — not `->`, not `⟶`
- Truncation: wrap in `<div className="min-w-0"><Text ... className="truncate min-w-0">...</Text></div>`
- Column `size` may increase from 160 to 220 to accommodate `A → B` format; adjust `minWidth` meta to `min-w-[180px]`

### Table — Refund Sub-line

Rendered inside the description cell (not a separate row or DataGrid grouping).

Structure:
```
<div className="min-w-0">
  <Text as="span" variant="body-sm" className="min-w-0 truncate font-semibold">
    {description}
  </Text>
  {/* only for refund rows with a linked original transaction */}
  {isRefund && refundOfTransaction && (
    <button
      type="button"
      className="mt-0.5 flex items-center gap-1 text-xs text-muted-foreground hover:text-foreground transition-colors"
      onClick={() => onOpenOriginal(refundOfTransaction.id)}
      aria-label={`View original transaction from ${formattedDate}`}
    >
      <span aria-hidden="true">↩</span>
      <span>{formattedDate} · {formatCurrency(refundOfTransaction.amountCents)}</span>
    </button>
  )}
</div>
```

- Glyph: `↩` (U+21A9 LEFTWARDS ARROW WITH HOOK)
- Separator: `·` (U+00B7 MIDDLE DOT)
- Date format: `MMM D, YYYY` (e.g. "Apr 3, 2025") — same locale formatting as existing date column
- Clicking the sub-line: opens the original transaction in `TransactionSheet` edit mode (calls `onEdit(originalTransactionId)`)
- `buildColumns` must accept an additional `onOpenOriginal: (id: string) => void` callback, or reuse the existing `onEdit` callback with a lookup from the `TransactionRow` array

### Table — Internal Filter Shortcut

Added to the filter bar in `TransactionFilterFields.tsx`.

- Filter field id: `typeGroup`
- Label: "Internal"
- Behavior: sets `type` URL param to a comma-separated value `transfer,settlement,contribution` OR uses a new `typeGroup=internal` query param — implementation choice left to executor (see Claude's Discretion)
- Visual: appears as a shortcut chip in the Filters bar, visually equivalent to a single-type filter chip
- When active: displays as "Type: Internal" in the chip
- Mutually exclusive with any individual type filter

API support required: `GET /api/transactions` must accept `type=transfer,settlement,contribution` (multiple values) or `typeGroup=internal`. The simpler approach is comma-separated `type` values parsed as an OR filter.

### Form — AssigneeSection (replaces SplitSection top-level)

New `AssigneeSection.tsx` component. The existing `SplitSection.tsx` and `AssigneeRow.tsx` are reused inside the "Customize split" Collapsible only.

**Toggle area:**

- One `Toggle` per org member, rendered in a `flex flex-wrap gap-2` row
- Each Toggle content: `<Avatar size="sm" /> <span className="ml-1.5 text-sm">{memberName}</span>`
- Toggle pressed state (`data-[state=on]`): `bg-primary/10 text-primary border-primary/30` (subtle fill, primary color text, faint primary border)
- Toggle unpressed state: default shadcn Toggle appearance (outline/ghost)
- Toggle min width: `min-w-[120px]` to prevent very narrow toggles for short names; `h-9`
- On press: add member to split; recalculate even split via LRM across all pressed members
- On un-press: remove member; recalculate even split across remaining pressed members

**Pressed defaults (D-17):**
- Single org member → auto-pressed (100% split) in create mode
- 2+ org members → all un-pressed in create mode

**Edit mode (D-18):**
- Pressed = member in `transaction.assignees` array; un-pressed = absent from array
- "Customize split" Collapsible opened if `assignees` has non-even distribution

**"Customize split" Collapsible:**

- Trigger: `<CollapsibleTrigger>` rendered as a text button — "Customize split ▾" / "Customize split ▴" using `ChevronDown`/`ChevronUp` (16px, `text-muted-foreground`)
- Trigger label: `Text variant="body-sm" className="text-muted-foreground hover:text-foreground"`
- Collapsed by default in create mode; expanded in edit mode if distribution is non-even
- `CollapsibleContent` contains the existing `SplitSection` rows (per-member % | $ rows + totals + global % | $ toggle)
- "Customize split" Collapsible only renders when at least one member toggle is pressed

**Component file:** `apps/web/src/components/transactions/AssigneeSection.tsx`

### Form — Locked Descriptions

`description` field rendering varies by type:

| Type | Description field state | Template |
|------|------------------------|----------|
| expense | Normal required Input | — |
| income | Normal required Input | — |
| refund (unlinked) | Normal required Input | — |
| refund (linked) | Locked | `Refund of {originalDescription}` |
| transfer | Locked | `Transfer from {fromAccountName} to {toAccountName}` |
| settlement | Locked | `Settlement — paid from {counterpartAccountName}` (or `Settlement` if no counterpart selected) |
| contribution | Locked | `{accountName} contribution` |

Template values update in real time as the user picks accounts (via `form.Subscribe` on `accountId` and `counterpartAccountId` field values).

Lock affordance:
- `Input` rendered with `readOnly` prop, `className="bg-muted text-muted-foreground cursor-default"`
- Left inset: `Lock` icon (lucide, 14px, `text-muted-foreground`) via `InputGroup` left slot or `relative` wrapper
- Right inset: "Edit" Button (`variant="ghost"`, `size="sm"`, `className="absolute right-2 top-1/2 -translate-y-1/2 h-auto py-0 px-1 text-xs"`)
- On "Edit" click: set local `isDescriptionUnlocked` state to `true` → re-render as normal `Input` with current value pre-filled; lock icon and Edit button hidden
- Unlock persists for the session (until form unmounts or type changes back to a locked type)
- On type change to a non-locked type: clear unlock state

### Form — Notes Field

- Field: `notes` (text nullable)
- Component: `Textarea` (or `Input` if single-line — use `Textarea` for multi-line; use `resize-none` with `rows={2}`)
- Label: "Notes"
- Placeholder: "Add a note…" (U+2026)
- `autocomplete="off"`
- Optional — no `FieldError` unless schema validation fires
- Positioning: bottom row of form, left slot of a `flex gap-4` row alongside `TransactionTagPicker` (right slot)
- Both fields: `className="flex-1"` — equal width split

### Form — Contribution Account Picker (D-20)

`ContributionFields.tsx` account Select renders each option as `{accountType} — {ownerMemberName}` (e.g. "TFSA — Emily").

- Uses existing `useGetAccounts()` data joined with org member display names
- `SelectItem` value: `account.id`
- `SelectItem` displayed label: `{account.type.toUpperCase()} — {account.ownerMemberName}` (em dash U+2014)
- Filter to accounts owned by a single member (investment accounts; exclude joint/household accounts)

### Form — Settlement "Paid from" (D-14, D-15)

`SettlementFields.tsx` `counterpartAccountId` field:

- Label: "Paid from"
- Shows tracked accounts only (all household accounts from `useGetAccounts()`)
- Each option: account name as rendered by existing account Select patterns
- Optional field — "(optional)" hint in `FieldLabel`
- `autocomplete="off"` on the underlying Select trigger

## Decisions Made

- **incomeTypeValues / investmentTypeValues as local const arrays** — not imported from `@ploutizo/db/schema/enums.ts`. The `@ploutizo/db` package is `apps/api`-only per CLAUDE.md package import boundary. Enum string values are copied as constants to keep `@ploutizo/validators` db-free.
- **updateTransactionSchema as flat partial** — using `baseTransactionSchema.partial().extend({...all type-specific optionals})` rather than a discriminated union partial. This allows PATCH-style updates where callers send only changed fields without knowing the original type; type-safety for the full object is enforced at read time (via createTransactionSchema).
- **TransactionFormSchema = createTransactionSchema** — direct alias preserves PascalCase form schema naming convention established in this package while avoiding duplication.

## Deviations from Plan

None — plan executed exactly as written.

## Threat Mitigations Applied

- **T-3.1-06** (datetime string accepted as date): `z.string().date()` used — Zod v4 `.date()` validates YYYY-MM-DD only, rejects ISO datetime strings. Test `'rejects ISO datetime as date'` confirms.
- **T-3.1-07** (amount=0 or negative): `z.number().int().positive()` used — rejects 0, negative, and fractional. Tests cover 0, -100, 1.5.
- **T-3.1-08** (discriminated union narrowing): `z.discriminatedUnion('type', [...])` used. Tests verify income rejects missing `incomeType`, transfer rejects missing `toAccountId`.

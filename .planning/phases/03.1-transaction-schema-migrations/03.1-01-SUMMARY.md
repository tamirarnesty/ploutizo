## Decisions Made

- Self-referential FK for `refundOf` uses `(): AnyPgColumn => transactions.id` lazy arrow — required by Drizzle to break circular type reference at declaration time
- `recurringTemplateId` stored as plain uuid with no `.references()` — v2 recurring templates table does not exist yet; reserved column as per D-07
- `percentage` on transactionAssignees explicitly documented as DISPLAY CACHE ONLY — Drizzle returns numeric as string, preventing accidental arithmetic in typed code
- `DATABASE_URL` sourced from `apps/api/.env` for migration apply — drizzle.config.ts uses `dotenv/config` which loads from cwd; worktree doesn't have node_modules so main repo drizzle-kit was used with PATH override

## Deviations from Plan

None - plan executed exactly as written. The migration generation required using the main repo's drizzle-kit binary with PATH override since the worktree has no node_modules, but this is an execution environment detail, not a plan deviation.

## Issues Encountered

- **Worktree has no node_modules**: `pnpm db:generate` failed with "command not found" in the worktree. Resolved by adding main repo's `node_modules/.bin` to PATH and running drizzle-kit directly from the worktree directory (picks up worktree's drizzle.config.ts via cwd).
- **DATABASE_URL not in root .env**: Found in `apps/api/.env`. Set as env var before running `drizzle-kit migrate`.

## User Setup Required

None - DATABASE_URL was already present in `apps/api/.env` and migration was applied successfully.

# Phase 4.2.1 Context

## Title

Clerk in-app auth route coverage and hosted URL elimination

## Why this phase exists

The app currently mixes in-app Clerk route handling with default Clerk-hosted fallbacks. This phase aligns all relevant Clerk flows to first-party app routes so users do not land on Clerk-hosted domains during normal usage.

## Included scope

- In-app routes for Clerk auth/profile/org management entry points
- Consistent Clerk component URL wiring to app routes
- Clerk dashboard redirect/path alignment for local and production domains
- Verification that users are returned to app routes for invite accept, auth, and profile flows

## Already completed and included

- API invite redirect hardening: `CLERK_INVITE_REDIRECT_URL` support in invitation payload (`redirect_url`)
- Env docs updated in `apps/api/.env.example`
- API tests covering `redirect_url` behavior

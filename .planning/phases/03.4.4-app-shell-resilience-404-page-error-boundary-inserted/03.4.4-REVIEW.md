## Warnings

### WR-01: `<a>` containing `<button>` — invalid HTML

**File:** `apps/web/src/components/not-found/NotFound.tsx:48-50`
**Issue:** `<Link to="/dashboard">` renders an `<a>` element. Placing `<Button>` (which renders a `<button>`) inside it produces a `<a><button>` nesting, which is forbidden by the HTML spec — interactive elements may not be nested. Browsers will mis-render or silently reorder the elements, and assistive technologies may behave incorrectly.

**Fix:** Use the `render` prop on `ButtonPrimitive` (or `asChild`-style composition) so the button's styles are applied to the `<a>` element, matching the pattern already used in `AppSidebar.tsx`:

```tsx
import { Link } from '@tanstack/react-router';
import { Button } from '@ploutizo/ui/components/button';

// Before (invalid):
<Link to="/dashboard">
  <Button>Go to Dashboard</Button>
</Link>

// After — Button renders the Link's <a> as its root element:
<Button render={<Link to="/dashboard" />}>
  Go to Dashboard
</Button>
```

Note: `Button` uses `@base-ui/react/button` which accepts a `render` prop for element substitution. This matches the pattern in `AppSidebar.tsx` (line 59).

### WR-02: `<a>` containing `<button>` — invalid HTML (ErrorBoundary)

**File:** `apps/web/src/components/error-boundary/ErrorBoundary.tsx:54-56`
**Issue:** Same root cause as WR-01. `<Link to="/dashboard">` wraps `<Button variant="outline">`, producing invalid `<a><button>` nesting.

**Fix:**
```tsx
// Before (invalid):
<Link to="/dashboard">
  <Button variant="outline">Go to Dashboard</Button>
</Link>

// After:
<Button variant="outline" render={<Link to="/dashboard" />}>
  Go to Dashboard
</Button>
```

## Layout Contract

Both pages share identical layout structure. Source: D-01, D-04, D-05, D-07, Onboarding reference pattern.

```
<div className="flex min-h-screen flex-col items-center justify-center bg-background px-6">
  <!-- Status bar — full width at top of content column -->
  <div> STATUS BAR </div>

  <!-- Art block — centered, opacity-40 -->
  <pre> DOT ART </pre>

  <!-- Error label -->
  <p> ERROR · 0x404  |  ERROR · 0x500 </p>

  <!-- Heading + subtext -->
  <Text as="h1" variant="h1"> ... </Text>
  <Text variant="body-sm" className="text-muted-foreground"> ... </Text>

  <!-- Action buttons -->
  <div className="flex gap-2"> ... </div>

  <!-- Dev-mode detail (DEV only) — see State Contract below -->
</div>
```

Content column max-width: `max-w-lg` (512px). Centered horizontally.

Status bar: full width of content column. Left-aligned `● STATUS — 404` / `● STATUS — ERROR`. Right-aligned path string. Use `flex items-center justify-between`. Border-bottom: `border-b border-border pb-3 mb-8`.

## State Contract

### 404 Page — States

| State | Description |
|-------|-------------|
| Default | Static page — always this state. No loading, no empty, no error sub-states. |

### Error Boundary — States

| State | Trigger | Rendering |
|-------|---------|-----------|
| Error (default) | `errorComponent` receives `ErrorComponentProps` | Full page layout with "ERR" art, heading, subtext, two action buttons |
| Dev detail — collapsed | `import.meta.env.DEV === true` | Monospace `error.message` line renders below buttons |
| Dev detail — expanded | User clicks "Show details" toggle | `<pre>` block with `error.stack` scrollable (max-height 200px, overflow-y auto) appears; toggle label changes to "Hide details" |

The "Show details" / "Hide details" toggle is rendered only when `import.meta.env.DEV` is true. In production, nothing below the action buttons is rendered.

`reset()` is called from `ErrorComponentProps` — no page reload, no additional confirmation.

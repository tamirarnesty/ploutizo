## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Plan acceptance criteria `grep -c "createSettlement" routes/settlements.ts == 1` was wrong**
- **Found during:** Task 2 verification
- **Issue:** The plan specified `== 1` but the import line + call site = 2 occurrences minimum. The plan's grep gate was mis-specified.
- **Fix:** Applied `>= 1` check instead. Actual count is 4 (import, type comment, call, export-related reference).
- **Files modified:** None (acceptance criteria fix, not code)

**2. [Rule 1 - Bug] Zod v4 UUID validator rejects synthetic test UUIDs**
- **Found during:** Task 2 testing (POST-SETTLE-01 returned 400 instead of 201)
- **Issue:** Zod v4 `.uuid()` validates RFC 4122 strictly — version bits must be `[1-8]`. Test UUIDs like `00000000-0000-0000-0000-000000000001` (version 0) were rejected.
- **Fix:** Changed all test UUIDs to RFC 4122 v4 compliant format (`550e8400-e29b-41d4-a716-44665544000x`).
- **Files modified:** `apps/api/src/__tests__/settlements.route.test.ts`, `apps/api/src/__tests__/settlements.service.test.ts`
- **Committed in:** `9029896` (part of Task 2 commit, iterative fix during test run)

**3. [Rule 1 - Bug] Double `await expect().rejects.toThrow()` pattern exhausted mockResolvedValueOnce**
- **Found during:** Task 2 testing (POST-SETTLE-07 assertion failed on second call)
- **Issue:** Two sequential `await expect(fn).rejects.toThrow()` calls trigger the async function twice. The first call consumed the `mockResolvedValueOnce` returning the mocked account; the second call got `undefined` (null fallback), throwing NotFoundError instead of DomainError.
- **Fix:** Changed to single-invocation pattern: `const err = await fn().catch(e => e)` + typed assertions.
- **Files modified:** `apps/api/src/__tests__/settlements.service.test.ts`
- **Committed in:** `9029896`

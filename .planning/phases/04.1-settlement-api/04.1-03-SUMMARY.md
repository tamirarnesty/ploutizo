## Test Isolation Guarantee

`settlements.service.test.ts` mocks ONLY `../lib/queries/settlements`. The service module (`../services/settlements`) is NOT mocked — the real `getSettlementBalances` implementation runs against mocked query rows. This guarantees GET-SETTLE-04/05/06/07 structurally exercise D-08 zero-balance omission, dueDate computation, and totalBalanceCents math via the actual service code.

`settlements.route.test.ts` mocks ONLY `../services/settlements`. Vitest's `vi.mock()` hoisting is isolated per file — splitting the files prevents the route-test mock from contaminating the service unit tests.

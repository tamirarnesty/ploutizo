# Phase 03.3: Transaction List UI - Context

**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Users can view all household transactions in a paginated, filterable DataGrid with server-side pagination, filtering, and sorting. Soft-delete individual transactions with undo. No create or edit forms (those ship in 03.4).

In-scope API work: patch `GET /api/transactions` to support type/category/account sort axes, and add `PATCH /api/transactions/:id/restore` for undo.

</domain>

<decisions>
## Implementation Decisions

### Filter State
- **D-01:** Filter state, pagination, and sort live in URL search params via TanStack Router `validateSearch`. Flat param names matching 03.2 API: `type`, `dateFrom`, `dateTo`, `accountId`, `categoryId`, `tagIds` (comma-separated), `assigneeId`, `page`, `limit`, `sort`, `order`.
- **D-02:** Default page size: 25 rows. User can change via DataGridPagination controls.
- **D-03:** Filter state updates URL immediately on ReUI Filters `onChange` — no submit/apply button.
- **D-04:** Clean URL at defaults — no params in URL when page=1, limit=25, sort=date, order=desc, no filters active.
- **D-05:** Default sort: `date desc` (newest first). Matches 03.2 API default.
- **D-06:** Sort controlled via DataGrid column header clicks (DataGridColumnHeader). Clicking a sortable column header writes `sort` + `order` to URL.
- **D-07:** Sortable columns: date, amount, type, category, account. Requires patching `apps/api/src/routes/transactions.ts` to add type/category/account `orderBy` cases — do this within Phase 03.3.
- **D-08:** Any filter change resets `page` to 1 in the URL.
- **D-09:** ReUI Filters component manages internal `Filter[]` state. On `onChange`, map Filter[] → flat URL search params via TanStack Router `navigate`. On mount, initialise Filters from URL search params (reverse mapping).

### Row Actions + Delete UX
- **D-10:** Rightmost column is an Actions column with a ⋮ `MoreHorizontal` icon button per row. Button visible on row hover only; always visible on touch/mobile (no hover state). Opens a dropdown.
- **D-11:** Dropdown contains: `Edit` (disabled, with tooltip "Available in next update") + `Delete`.
- **D-12:** No row click action in 03.3. Row click does nothing. Edit wired in 03.4.
- **D-13:** Clicking Delete opens a confirm Dialog: "Delete this transaction? This can't be undone." with Cancel + Delete buttons.
- **D-14:** After confirmed delete: optimistically remove from list, call `DELETE /api/transactions/:id`, then show toast with "Transaction deleted" + Undo action (5-second window). Clicking Undo calls `PATCH /api/transactions/:id/restore`. `onSettled` invalidates `["transactions"]` query key in both cases.
- **D-15:** Add `PATCH /api/transactions/:id/restore` to `apps/api/src/routes/transactions.ts` in this phase. Sets `deleted_at = null`. Same tenant guard as delete.

### Column Set + Amount Display
- **D-16:** Columns (left to right): Date | Type | Description | Category | Account | Assignees | Tags | Amount | Actions.
- **D-17:** Amount column: colored by transaction type, unsigned cents formatted via `formatCurrency`. Expense / Refund / Settlement = red (`text-destructive` or `text-red-600`). Income / Contribution = green (`text-green-600`). Transfer = muted neutral.
- **D-18:** Assignees column: shadcn `Avatar` + `AvatarGroup` with count (initials fallback if no `imageUrl`). Show max 3 avatars, then `+N` overflow. Reference: https://ui.shadcn.com/docs/components/base/avatar#avatar-group-count
- **D-19:** Type column: shadcn `Badge` with per-type color variants. Define consistent color map: expense=destructive/red, income=green, transfer=secondary/gray, settlement=blue, refund=orange, contribution=purple.
- **D-20:** Tags column: tag names as small pill badges. Show first 2, then `+N` overflow text. Tags column omitted (shown as `—`) if no tags exist on the transaction.
- **D-21:** Category column: Lucide icon (from `categoryIcon` string) + category name. Show `—` for types that have no category (income, transfer, settlement, contribution).
- **D-22:** Fix `formatCurrency` in `apps/web/src/lib/formatCurrency.ts` to use `currency: 'CAD'` (currently incorrectly set to `'USD'`). First phase to render amounts in the UI.
- **D-23:** Page heading: `Transactions` (h1). Top-right CTA: `Add transaction` button — present in 03.3 but disabled/grayed out with tooltip "Create transactions coming soon". Button wired in 03.4.
- **D-24:** Empty state (no transactions): centered inside the DataGrid area — "No transactions yet" + subtext + `Add transaction` button (also disabled in 03.3). Matches AccountsTable empty state pattern.
- **D-25:** Empty state (active filters, no results): "No transactions match your filters" + `Clear filters` link.

### Filter Bar
- **D-26:** Use ReUI `Filters` component (`packages/ui/src/components/reui/filters.tsx` — needs to be installed from the ReUI registry as part of this phase).
- **D-27:** Filter bar sits between the page header and the DataGrid. Full-width, no sticky behavior.
- **D-28:** 6 filter fields in `FilterFieldConfig`:
  - `type` — single select (one type at a time; matches 03.2 API `?type=` single value)
  - `dateRange` — date range picker with `between` operator → maps to `dateFrom` + `dateTo` URL params
  - `accountId` — single select; options populated from `useGetAccounts()` query
  - `categoryId` — single select; options populated from categories query
  - `assigneeId` — single select; options populated from org members query
  - `tagIds` — multiselect; options populated from tags query; OR logic (any tag match)
- **D-29:** Reference the ReUI DataGrid + Filters integration example for the `FilterFieldConfig` structure and how to wire `onChange` to server-side filtering.

### Claude's Discretion
- Exact Drizzle `orderBy` clause additions for type/category/account sort in the API patch
- How to derive initials for Avatar fallback when `imageUrl` is null
- Skeleton placeholder shapes for each column during loading
- Tag pill color styling (use tag `colour` field from API response if present, else default)
- Exact Tailwind classes for amount color variants (use design system tokens where possible)
- DataGridColumnHeader click-to-sort wiring detail
- Whether to debounce URL updates from date range inputs to avoid excessive API calls

</decisions>

## Color

All tokens are CSS variables from `packages/ui/src/styles/globals.css`. Use Tailwind semantic classes, not raw hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` | Page background, DataGrid surface |
| Secondary (30%) | `bg-card` / `bg-muted` | Filter bar background, empty state container, skeleton fills |
| Accent (10%) | `text-primary` / `bg-primary` | "Add transaction" button (disabled state in 03.3 — use `opacity-50` on the button, not a separate accent use) |

Accent reserved for: the "Add transaction" button CTA only. No other element uses `primary` color in this phase.

## Interaction States

### Row Actions Dropdown (source: D-10, D-11, D-12, D-13)

```
⋮ button (MoreHorizontal, 16px icon, 44×44px touch target)
└── DropdownMenu
    ├── Edit          [disabled]  → Tooltip: "Available in next update"
    └── Delete        [enabled]   → Opens confirm Dialog
```

### Delete Confirmation Dialog (source: D-13, D-14)

- Title: "Delete transaction?"
- Body: "This action cannot be undone."
- Footer: `Cancel` (secondary) | `Delete` (destructive variant)
- On confirm: optimistic remove → `DELETE /api/transactions/:id` → toast with Undo (5s)

### Undo Toast (source: D-14)

- Message: "Transaction deleted"
- Action button: "Undo" → calls `PATCH /api/transactions/:id/restore`
- Duration: 5000ms
- `onSettled` in both delete and restore mutations: invalidate `["transactions"]`

### Disabled CTA (source: D-23)

- "Add transaction" button: `disabled` + `title="Create transactions coming soon"` (or Tooltip component)
- Visual: opacity reduced via `disabled:opacity-50 disabled:cursor-not-allowed` (shadcn Button default)

## Filter Bar (source: D-26 through D-29)

Component: ReUI `Filters` — install from `@reui/filters` registry.

Position: between page heading row and DataGrid, full-width, no sticky.

| Filter Field | Field ID | Operator | Input Type | Data Source |
|-------------|----------|----------|------------|-------------|
| Type | `type` | `equals` | Select (single) | Static: expense, income, transfer, settlement, refund, contribution |
| Date Range | `dateRange` | `between` | Date range picker | — (maps to `dateFrom` + `dateTo` URL params) |
| Account | `accountId` | `equals` | Select (single) | `useGetAccounts()` |
| Category | `categoryId` | `equals` | Select (single) | Categories query |
| Assignee | `assigneeId` | `equals` | Select (single) | Org members query |
| Tags | `tagIds` | `in` | Multi-select | Tags query |

URL param behavior (source: D-01, D-03, D-08, D-09):
- Updates URL immediately on `onChange` — no submit button.
- Any filter change resets `page` to 1.
- Clean URL at defaults (no params when all filters cleared, page=1, limit=25, sort=date, order=desc).

## Accessibility Contract

- All icon-only buttons (⋮ Actions) must have `aria-label="Transaction actions"`.
- Disabled "Add transaction" button: `aria-disabled="true"` + tooltip via `title` or Tooltip component.
- Delete Dialog: focus trapped inside while open; confirm button receives focus on open.
- Sortable column headers: `DataGridColumnHeader` supplies `aria-sort` attribute — do not override.
- Avatar initials fallback: `aria-label="{displayName}"` on each Avatar.
- AvatarGroup overflow "+N" text: `aria-label="and {N} more"`.
- Tag pills: render as `<span>` (not interactive); no ARIA role needed.
- Toast Undo button: keyboard accessible; `role="status"` on toast container (Sonner default).

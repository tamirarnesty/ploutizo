## Summary

Installed the ReUI Filters component into `packages/ui` and created all three data-access hooks for transactions.

## What Was Built

**ReUI Filters component** — installed at `packages/ui/src/components/reui/filters.tsx` via `npx shadcn add @reui/filters`. Exports:
- `Filter<T>` — `{ id, field, operator, values: T[] }` — the filter state atom
- `FilterFieldConfig<T>` — field definition (type, label, operators, options)
- `FilterFieldsConfig<T>` — `FilterFieldConfig<T>[] | FilterFieldGroup<T>[]`
- `Filters<T>` component props: `filters: Filter<T>[]`, `fields: FilterFieldsConfig<T>`, `onChange: (filters: Filter<T>[]) => void`
- `between` operator: values are `T[]` with 2 elements `[from, to]` (array, not `{from, to}` object)
- `isAnyOf` / `is` operators: values are `T[]` (string array for multi-select)

**Data-access hooks** — `apps/web/src/lib/data-access/transactions/`:

| File | Export | Key behavior |
|------|--------|-------------|
| `useGetTransactions.ts` | `useGetTransactions(params)`, `fetchTransactions(params)`, `TransactionRow`, `TransactionQueryParams`, `TransactionListResponse` | Builds full query string with all 8 params (page, limit, sort, order, type, dateFrom, dateTo, accountId, categoryId, assigneeId, tagIds as comma-separated) |
| `useDeleteTransaction.ts` | `useDeleteTransaction()` | `onMutate` → `cancelQueries` + `getQueriesData` snapshot + `setQueriesData` optimistic remove; `onError` → full rollback; `onSettled` → invalidate `['transactions']` |
| `useRestoreTransaction.ts` | `useRestoreTransaction()` | `PATCH /api/transactions/:id/restore`; `onSettled` → invalidate `['transactions']` |
| `index.ts` | Re-exports all three | Barrel export |

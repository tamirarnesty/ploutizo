### Pattern 1: TanStack Router `validateSearch` for Filter State
**What:** Route-level schema defines and coerces all search params. Component reads via `useSearch`, writes via `useNavigate`.
**When to use:** Any route with URL-driven filter/sort/pagination state.

```typescript
// Source: [VERIFIED: apps/web/src/routes/_layout.accounts.tsx pattern + TanStack Router docs]
// apps/web/src/routes/_layout.transactions.tsx
import { createFileRoute, useNavigate, useSearch } from '@tanstack/react-router'
import { z } from 'zod'

const transactionSearchSchema = z.object({
  page: z.number().int().min(1).catch(1),
  limit: z.number().int().min(1).max(200).catch(25),
  sort: z.enum(['date', 'amount', 'type', 'category', 'account']).catch('date'),
  order: z.enum(['asc', 'desc']).catch('desc'),
  type: z.string().optional(),
  dateFrom: z.string().optional(),
  dateTo: z.string().optional(),
  accountId: z.string().optional(),
  categoryId: z.string().optional(),
  assigneeId: z.string().optional(),
  tagIds: z.string().optional(), // comma-separated UUIDs
})

export const Route = createFileRoute('/_layout/transactions')({
  validateSearch: (search) => transactionSearchSchema.parse(search),
  component: Transactions,
})
```

**Key detail:** `useSearch({ from: '/_layout/transactions' })` returns the parsed/coerced params. `useNavigate({ from: '/_layout/transactions' })` with `search: (prev) => ({ ...prev, ...updates })` updates params while preserving others. [ASSUMED — exact API for TanStack Router v1.168]

**Clean URL (D-04):** Use `navigate` with `replace: true` and only include params that differ from defaults. Alternatively, strip default values in a helper before calling navigate.

### Pattern 2: `useReactTable` with `manualPagination`
**What:** Pass external row count + page state to TanStack Table; disable internal pagination model.
**When to use:** Server-side pagination (this phase).

```typescript
// Source: [VERIFIED: apps/web/src/components/accounts/AccountsTable.tsx — adapted for manualPagination]
const table = useReactTable({
  data: transactions,
  columns,
  getCoreRowModel: getCoreRowModel(),
  manualPagination: true,           // disables getPaginationRowModel
  rowCount: totalCount,             // from API response envelope
  state: {
    pagination: { pageIndex: page - 1, pageSize: limit },
    sorting: [{ id: sort, desc: order === 'desc' }],
  },
  onPaginationChange: (updater) => {
    const next = typeof updater === 'function'
      ? updater({ pageIndex: page - 1, pageSize: limit })
      : updater
    navigate({ search: (prev) => ({ ...prev, page: next.pageIndex + 1, limit: next.pageSize }) })
  },
  onSortingChange: (updater) => {
    const next = typeof updater === 'function' ? updater([{ id: sort, desc: order === 'desc' }]) : updater
    const col = next[0]
    if (!col) return
    navigate({ search: (prev) => ({ ...prev, sort: col.id, order: col.desc ? 'desc' : 'asc', page: 1 }) })
  },
  manualSorting: true,
})
```

### Anti-Patterns to Avoid
- **`getPaginationRowModel()` with server-side pagination:** Drop it; use `manualPagination: true` + `rowCount` instead. AccountsTable uses client-side pagination — TransactionsTable must not copy this part.
- **`onSuccess` for invalidation:** Use `onSettled` — `onSuccess` doesn't fire on error, leaving stale data.
- **Inline component definitions in column `cell` renderers:** Define cell renderer functions outside the `useMemo` columns array, or use stable inline components (they re-run but don't cause child unmount/remount issues because `useMemo` deps include the stable column array).
- **Raw `fetch()` in data-access hooks:** Always use `apiFetch` from `@/lib/queryClient`.
- **`forwardRef` in new components:** React 19 project — pass `ref` as a regular prop.

## Common Pitfalls

### Pitfall 1: `validateSearch` `catch()` vs `default()`
**What goes wrong:** Using `.default(1)` instead of `.catch(1)` in the Zod schema — `.default()` only applies when the field is absent; `.catch()` applies when the value is absent OR invalid (e.g., `?page=foo`).
**Why it happens:** Zod `.default()` seems equivalent but doesn't handle invalid values.
**How to avoid:** Use `.catch()` for all URL search param fields so invalid/missing values always fall back to defaults.
**Warning signs:** Router throws validation errors when navigating with partial/invalid search params.

### Pitfall 2: `useSearch` return type mismatch
**What goes wrong:** `useSearch` returns `unknown` without the `from` option, causing TypeScript errors in component.
**How to avoid:** Always call `useSearch({ from: '/_layout/transactions' })` — the `from` option narrows the return type to the route's validated search schema.

### Pitfall 3: DataGrid `recordCount` vs `rowCount`
**What goes wrong:** Passing `rowCount` to `useReactTable` (TanStack Table prop) but confusing it with `recordCount` on the `<DataGrid>` component prop. These are separate: `rowCount` is TanStack Table's server-side total for page math; `recordCount` is what ReUI DataGrid displays in the pagination info string.
**How to avoid:** Pass both: `rowCount` to `useReactTable` config, `recordCount={total}` to `<DataGrid>`.

### Pitfall 4: Filter `tagIds` comma-serialization
**What goes wrong:** `tagIds` URL param is comma-separated (`?tagIds=a,b,c`). TanStack Router's `validateSearch` receives it as a single string. The data-access hook must split on comma before passing to `apiFetch`.
**How to avoid:** In `useGetTransactions`, split `tagIds` string to array before building the query string: `tagIds.split(',').filter(Boolean)`.

### Pitfall 5: Clean URL at defaults
**What goes wrong:** Every navigation call includes all params, including defaults (e.g., `page=1&limit=25&sort=date&order=desc`), creating noise in URLs and breaking bookmark sharing.
**How to avoid:** In the navigate call, strip params that equal their defaults. Write a `buildTransactionSearch` helper that omits default values.

### Pitfall 6: `DataGridColumnHeader` sorting requires `enableSorting` on column def
**What goes wrong:** `DataGridColumnHeader` checks `column.getCanSort()` to show the sort icon and wiring. If column def doesn't set `enableSorting: true`, the header renders as a non-interactive label.
**How to avoid:** Set `enableSorting: true` on each sortable column def (date, amount, type, category, account). [VERIFIED: DataGridColumnHeader source — `canSort = column.getCanSort()`]

### Pitfall 7: Amount sorting column ID must match `sort` param values
**What goes wrong:** TanStack Table column `id` (e.g., `'amount'`) must match the `sort` param value sent to the API. If the column id is `'amount'` but the API accepts `sort=amount`, they must be the same string.
**How to avoid:** Use the API's accepted sort field names as the column `id` values: `date`, `amount`, `type`, `category`, `account`.

### Pitfall 8: `manualSorting: true` required alongside `manualPagination: true`
**What goes wrong:** Without `manualSorting: true`, TanStack Table performs client-side sorting on the already-paginated data, re-ordering just the current page instead of triggering a new server fetch.
**How to avoid:** Set both `manualPagination: true` and `manualSorting: true`.

### Pitfall 9: ReUI Filters Tailwind v4 compatibility
**What goes wrong:** STATE.md notes "ReUI Filters not yet tested" with Tailwind v4 — DataGrid works but Filters is unconfirmed.
**How to avoid:** After installing, inspect rendered output. If any bare `border` or `ring` classes produce unexpected color, add explicit color classes at the usage site via `className` prop override (CLAUDE.md base component override pattern).

### Restore Query (API)
```typescript
// Source: [VERIFIED: apps/api/src/lib/queries/transactions.ts softDeleteTransactionQuery pattern]
export async function restoreTransactionQuery(
  id: string,
  orgId: string
): Promise<{ id: string } | null> {
  const rows = await db
    .update(transactions)
    .set({ deletedAt: null, updatedAt: new Date() })
    .where(
      and(
        eq(transactions.id, id),
        eq(transactions.orgId, orgId),
        isNotNull(transactions.deletedAt)  // only restore actually-deleted rows
      )
    )
    .returning({ id: transactions.id })
  return rows.at(0) ?? null
}
```

## Security Domain

### Applicable ASVS Categories

| ASVS Category | Applies | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | yes (API) | Clerk `getAuth(c)` — already on all routes |
| V3 Session Management | no | Stateless JWT; handled by Clerk |
| V4 Access Control | yes | `tenantGuard()` middleware + `orgId` WHERE clause on all queries |
| V5 Input Validation | yes | Zod on restore route; TanStack Router `validateSearch` on frontend |
| V6 Cryptography | no | No crypto in this phase |

### Known Threat Patterns

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| IDOR on restore endpoint | Elevation of privilege | `eq(transactions.orgId, orgId)` in `restoreTransactionQuery` WHERE clause |
| Restoring another org's deleted transaction | Elevation of privilege | Same — `orgId` guard in WHERE |
| `tagIds` comma-injection | Tampering | Split on comma, validate each UUID format; invalid UUIDs produce no DB match (UUID FK constraint) |

## Project Constraints (from CLAUDE.md)

- React SPA — TanStack Start/Router. No RSC, no `"use server"`, no `next/*`.
- All data fetching in `apps/web` via TanStack Query hooks in `apps/web/src/lib/data-access/`. No raw `fetch()` in components.
- All API requests from `apps/web` go through `apiFetch` in `apps/web/src/lib/queryClient.ts`.
- Never modify `packages/ui/src/components/reui/*` or shadcn-generated files. Override at usage site via `className`.
- Test runner: `pnpm test` or `pnpm --filter <package> test` — never `npx vitest` directly.
- **vercel-react-best-practices** skill applies: all queries fired at top level (no waterfall), `useQueries` for parallel, `useCallback`/`useMemo` for stable refs, no inline component definitions in render.
- **vercel-composition-patterns** skill applies: no boolean props for mode switches; React 19 — no `forwardRef`.
- **web-design-guidelines** skill applies: `autocomplete` on inputs, `min-w-0` on truncating flex children, `aria-label` on icon-only buttons, Unicode `—` for empty cells.

**SwatchButton component spec** (from D-15/D-16):
```tsx
// apps/web/src/components/categories/SwatchButton.tsx
// Wraps a raw <button> — NOT shadcn Button. Incompatible with Button base styles.
// Preserves all swatch-specific accessibility: role="radio", aria-checked, aria-label.

interface SwatchButtonProps {
  colour: string   // Tailwind bg-* class, e.g. "bg-slate-500"
  name: string     // aria-label value
  checked: boolean // aria-checked value
  onClick: () => void
}

export const SwatchButton = ({ colour, name, checked, onClick }: SwatchButtonProps) => (
  <button
    type="button"
    role="radio"
    aria-checked={checked}
    aria-label={name}
    onClick={onClick}
    className={[
      `size-6 rounded ${colour} transition-all`,
      checked ? "ring-2 ring-primary ring-offset-2" : "hover:scale-110",
    ].join(" ")}
  />
)
```

**Key constraints:**
- No imports needed (no `cn()` — string array join is already used in the original)
- No shadcn `Button` import — this is intentionally a raw `<button>`
- Props mirror exactly the values ColourPicker already passes inline

## Shared Patterns

### Text import
**Apply to:** Every migrated file that uses `<Text>`.
```tsx
import { Text } from "@ploutizo/ui/components/text"
```

### Heading migration pattern
**Source:** CONTEXT.md D-03, text.tsx variant `h3` (line 11)
**Apply to:** All files with `<h1>`–`<h6>` raw elements
```tsx
// h3 variant classes: "font-heading text-xl font-semibold leading-snug"
<Text as="h2" variant="h3">Section Title</Text>
// With layout overrides:
<Text as="h1" variant="h3" className="mb-4">Page Title</Text>
```

### Muted body-sm paragraph pattern
**Source:** CONTEXT.md D-06
**Apply to:** All `<p className="text-sm text-muted-foreground">` elements
```tsx
<Text variant="body-sm" className="text-muted-foreground">Description text.</Text>
```

### Caption (xs muted) pattern
**Source:** text.tsx `caption` variant — `"text-xs text-muted-foreground leading-normal"` (line 14)
**Apply to:** All `<p className="text-xs text-muted-foreground">` elements
```tsx
<Text variant="caption">Helper text.</Text>
// caption variant already includes text-muted-foreground — no className override needed
```

### Error text pattern
**Source:** CONTEXT.md D-07/D-08 (new `error` variant to be added)
**Apply to:** All `<p className="text-xs text-destructive">` elements
```tsx
// Requires error variant in text.tsx to be added first (Wave 0 dependency)
<Text variant="error">{errorMessage}</Text>
```

### Text-content span pattern
**Source:** CONTEXT.md D-10, MemberRow.tsx line 31, TransactionColumns.tsx lines 64, 153
**Apply to:** All `<span>` with direct text content and typography classes
```tsx
// Inline text in flex rows — always include as="span"
<Text as="span" variant="body-sm" className="text-muted-foreground">{value}</Text>
<Text as="span" variant="caption">{smallText}</Text>
```

### Structural span → div pattern
**Source:** CONTEXT.md D-11, MemberRow.tsx lines 24, 34
**Apply to:** Flex/block container `<span>` elements with no direct text content
```tsx
// Before: <span className="flex min-w-0 items-center gap-2">
// After:
<div className="flex min-w-0 items-center gap-2">
```

### Exemptions — spans that stay as raw `<span>`
**Source:** CONTEXT.md D-13
**Apply to:** Never convert these
- `<span>` inside `<FieldLabel>` / shadcn primitive children (RuleForm lines 163/180, AccountForm lines 263/285)
- `<span className="text-muted-foreground">` inside `<SelectValue>` (RuleForm line 193)
- Purely decorative/structural icon wrappers with no text (CategoriesSettings lines 137, 140)

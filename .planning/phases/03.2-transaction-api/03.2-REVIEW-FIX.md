## Fixed Issues

### WR-01: NaN propagates through pagination clamp for non-numeric query params

**Files modified:** `apps/api/src/routes/transactions.ts`
**Commit:** 1ffb2b9
**Applied fix:** Replaced `Number(c.req.query(...) ?? default)` with explicit `Number.isFinite` guards. Non-finite values fall back to defaults (page=1, limit=50) instead of propagating NaN into the query layer.

---

### WR-02: PATCH split-sum validation skipped when assignees are patched without amount

**Files modified:** `apps/api/src/services/transactions.ts`
**Commit:** 0ad3a4d
**Applied fix:** When `data.assignees` is present but `data.amount` is absent, the stored transaction amount is fetched via `fetchTransactionById` and used as `amountToCheck`. Split validation now always runs when assignees are being replaced.

---

### WR-03: `refundOf` ownership check only fires for `type === 'refund'`

**Files modified:** `apps/api/src/routes/transactions.ts`
**Commit:** e5a4bf0 (amended into 932ba0b)
**Applied fix:** Changed `if (data.type === 'refund' && data.refundOf)` to `if ('refundOf' in data && data.refundOf)`. The `in` narrowing is required because `refundOf` only exists on the `refund` variant of the discriminated union — direct property access causes a TS2339 type error. The schema still enforces the constraint; this makes the route-level check independent of type.

---

### WR-04: `tx as unknown as DrizzleTransaction` cast hides a type mismatch

**Files modified:** `apps/api/src/lib/queries/transactions.ts`, `apps/api/src/services/transactions.ts`
**Commit:** 932ba0b
**Applied fix:** Replaced the manual `PgTransaction<NeonQueryResultHKT, Record<string, never>, ...>` construction with `type DrizzleTransaction = Parameters<Parameters<typeof db.transaction>[0]>[0]`. This derives the type directly from the actual `db` instance (which carries the full schema), eliminating the schema mismatch. The three `tx as unknown as DrizzleTransaction` casts in `services/transactions.ts` were removed entirely, and the now-unused `DrizzleTransaction` import in that file was cleaned up.

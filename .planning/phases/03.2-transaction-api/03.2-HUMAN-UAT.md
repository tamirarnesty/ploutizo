### 2. Soft-delete end-to-end

expected: DELETE /api/transactions/:id sets deletedAt; subsequent GET /api/transactions/:id returns 404; GET /api/transactions (list) excludes the deleted row
result: passed

```bash
# Step 1: Create a transaction to delete
export DELETE_TXN_ID=$(curl -s -X POST "$BASE/api/transactions" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "expense",
    "accountId": "'"$ACCOUNT_ID"'",
    "amount": 999,
    "date": "2026-04-10",
    "description": "UAT soft-delete test"
  }' | jq -r '.data.id')
echo "Created: $DELETE_TXN_ID"

# Step 2: Soft-delete it
curl -s -X DELETE "$BASE/api/transactions/$DELETE_TXN_ID" \
  -H "Authorization: Bearer $TOKEN" | jq .
# Expected: 200, body.data.id === $DELETE_TXN_ID

# Step 3: GET by ID → 404
curl -s "$BASE/api/transactions/$DELETE_TXN_ID" \
  -H "Authorization: Bearer $TOKEN" | jq .
# Expected: 404, body.error.code === "NOT_FOUND"

# Step 4: GET list → deleted row excluded (search by date to narrow results)
curl -s "$BASE/api/transactions?dateFrom=2026-04-10&dateTo=2026-04-10" \
  -H "Authorization: Bearer $TOKEN" | jq '.data | map(select(.id == "'"$DELETE_TXN_ID"'"))'
# Expected: []

# Step 5: Second DELETE → 404
curl -s -X DELETE "$BASE/api/transactions/$DELETE_TXN_ID" \
  -H "Authorization: Bearer $TOKEN" | jq .
# Expected: 404, body.error.code === "NOT_FOUND"
```

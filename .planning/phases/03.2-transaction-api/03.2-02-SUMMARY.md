## What Was Built

**lib/helpers.ts** — `badRequest()` shared 400 helper (D-09).

**lib/queries/transactions.ts** — All Drizzle SQL:
- `buildConditions()` — composable WHERE array for list + count
- `buildListQuery()` — paginated list with category + account left joins
- `countQuery()` — total for pagination envelope
- `fetchTransactionById()` — single with joins + soft-delete guard
- `enrichTransactions()` — parallel assignee+tag sub-queries (avoids cartesian)
- `refundOfExists()` — org-scoped refundOf validation query
- `softDeleteTransactionQuery()` — sets deletedAt, returns null if not found
- `updateTransactionScalarsQuery()` — inside-transaction scalar update
- `replaceAssignees()` / `replaceTags()` — delete-then-insert inside transaction

**services/transactions.ts** — Zero drizzle-orm imports:
- `validateSplitSum()` — pure function, sum check before DB write
- `checkRefundOfOwnership()` — delegates to refundOfExists()
- `createTransaction()` — db.transaction() with optional assignees+tags
- `listTransactions()` — parallel base+count, then enrichTransactions()
- `getTransaction()` — fetchById + enrich single

**routes/transactions.ts** — Thin HTTP handlers, no db/drizzle imports:
- POST /: Zod (D-10) → split sum (D-11) → refundOf ownership (D-13) → createTransaction → 201
- GET /: query param parsing (tagIds dual-format) → listTransactions → 200
- GET /:id: getTransaction → 200 or NOT_FOUND 404

## Deviations from Plan

**1. [Rule 1 - Bug] Zod v4 strict UUID validation in test payloads**
- **Found during:** Task 2 test execution
- **Issue:** Zod v4 uses stricter UUID regex (`[1-8]` for version bits, `[89abAB]` for variant bits). Placeholder UUIDs like `00000000-0000-0000-0000-000000000099` fail validation — only `00000000-0000-0000-0000-000000000000` (nil UUID) is an allowed exception.
- **Fix:** Replaced placeholder UUIDs in test payloads with proper v4 UUIDs (`a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11` pattern).
- **Files modified:** `apps/api/src/__tests__/transactions.test.ts`
- **Commit:** c355c23

## Split input contract

| Option | Description | Selected |
|--------|-------------|----------|
| Client sends pre-calc amounts | `assignees: [{memberId, amountCents, percentage}]`; API validates sum == amount; split math in 03.4 UI | ✓ |
| API computes from memberIds | Client sends memberIds; server runs Largest Remainder Method; changes assigneeSchema | |
| Support both | Both code paths; more flexible for programmatic use | |

**User's choice:** Client sends pre-calculated amounts (recommended)
**Notes:** 03.4 UI is responsible for running Largest Remainder Method before submit. Keeps API simple. Existing `assigneeSchema` in validators already reflects this contract (amountCents required).

## Filter + pagination

| Option | Description | Selected |
|--------|-------------|----------|
| Full filter + pagination in 03.2 | All filter params + offset pagination in this phase | ✓ |
| Basic list in 03.2, filters in 03.3 | Defer filtering query logic to UI phase | |

**User's choice:** Full filter + pagination in 03.2 (recommended)

| Option | Description | Selected |
|--------|-------------|----------|
| Offset pagination | ?page=1&limit=50; returns {data, total, page, limit} | ✓ |
| Cursor-based | ?cursor=lastId; better for infinite scroll | |

**User's choice:** Offset (recommended)

| Option | Description | Selected |
|--------|-------------|----------|
| Configurable sort (date + amount) | ?sort=date\|amount&order=asc\|desc; default date desc | ✓ |
| Fixed date desc | Always newest first; simplest | |

**User's choice:** Configurable sort with default descending
**Notes:** Allows 03.3 DataGrid columns to be sortable server-side, not just client-side. Fields: date and amount. Default: sort=date&order=desc.

| Option | Description | Selected |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] transactions.ts inline BAD_REQUEST instead of throwing DomainError**
- **Found during:** Task 2 analysis
- **Issue:** Plan template says `throw new DomainError(400, splitError)` for split validation, but the transactions test mounts `transactionsRouter` directly on a plain `new Hono()` without `app.onError()`. Throwing DomainError without an onError handler results in Hono returning 500, but `TXN-POST-04` expects 400 with `code: 'BAD_REQUEST'`.
- **Fix:** Split validation and PATCH catch use `return c.json({ error: { code: 'BAD_REQUEST', message } }, 400)` inline — identical behavior to the deleted `badRequest()` helper. This preserves all test expectations while removing the `badRequest` import.
- **Files modified:** apps/api/src/routes/transactions.ts
- **Commit:** bb25478

**2. [Rule 3 - Blocking] pnpm-lock.yaml duplicate mapping key**
- **Found during:** Task 1 setup
- **Issue:** Worktree's `pnpm-lock.yaml` had a duplicate mapping key for `@clerk/backend@3.2.9` (from prior wave merge conflicts), causing `pnpm install --frozen-lockfile` to fail. `pnpm install` regenerated the lockfile correctly.
- **Fix:** Ran `pnpm install` (non-frozen) to repair the lockfile.
- **Files modified:** pnpm-lock.yaml
- **Commit:** bb25478

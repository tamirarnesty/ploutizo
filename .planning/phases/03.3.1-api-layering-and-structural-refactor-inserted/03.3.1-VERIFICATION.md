### Notable Deviations (Auto-fixed by executor — improvements)

1. **DomainError.code optional field** — `lib/errors.ts` has an optional `code?: string` parameter added to DomainError constructor. This is an improvement over the plan spec: allows services to pass machine-readable codes (e.g., `'INVALID_REGEX'`), and `index.ts` onError uses `err.code ?? 'DOMAIN_ERROR'`. `NotFoundError` passes `'NOT_FOUND'` as its code.

2. **transactions.ts inline BAD_REQUEST** — Split validation and PATCH error handling use inline `c.json({ error: { code: 'BAD_REQUEST' } }, 400)` rather than `throw new DomainError(400, ...)`. Justified: the transactions test mounts the router directly without `onError`, so throwing DomainError would produce 500. Behavior is functionally identical.

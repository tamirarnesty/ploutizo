## Fixed Issues

### CR-01: Hard-coded role mapping collapses all members to `admin`

**Files modified:** `apps/api/src/services/webhooks.ts`
**Commit:** d28ed66
**Applied fix:** Changed ternary from `'admin' : 'admin'` to `'admin' : 'member'` so non-admin org members are stored with the `member` role.

### WR-01: Partial regex validation — update path can save an invalid regex

**Files modified:** `apps/api/src/services/merchant-rules.ts`
**Commit:** 28a878f
**Applied fix:** Replaced the `matchType && pattern` guard with a standalone `data.pattern !== undefined` check that always attempts `new RegExp(pattern)` compilation, regardless of whether `matchType` is in the patch payload.

### WR-02: `updateAccount` strips type safety with `as Record<string, unknown>`

**Files modified:** `apps/api/src/services/accounts.ts`
**Commit:** 7c99ca7
**Applied fix:** Destructured `archivedAt` out of `data` alongside `memberIds` (archival has its own dedicated endpoint and the schema's `archivedAt` is a string, not `Date`). The remaining `updateData` is correctly typed as `Partial<typeof accounts.$inferInsert>` and passed to the query without any cast. Confirmed `npx tsc --noEmit` produces zero errors on the file.

### WR-03: `DomainError` message/code arguments are swapped in households service

**Files modified:** `apps/api/src/services/households.ts`, `apps/api/src/routes/households.ts`
**Commit:** 529633f
**Applied fix:** Added the machine-readable error code as the proper third argument to all six `DomainError` calls in the service (inviteMember + removeMember paths), with a human-readable string as the second argument. Updated both route `catch` blocks to read `err.code ?? 'DOMAIN_ERROR'` instead of `err.message`. Removed the stale comment that documented the inverted convention.

### WR-04: `archiveAccount` query does not set `updatedAt` (categories variant)

**Files modified:** `apps/api/src/lib/queries/categories.ts`
**Commit:** 26826c6
**Applied fix:** Added `updatedAt: new Date()` to the `.set()` call in `archiveCategory`. (Note: `archiveAccount` in `accounts.ts` already stamped `updatedAt` — the review finding was accurate only for the categories file.)

### WR-05: `updateCategory` query does not set `updatedAt`

**Files modified:** `apps/api/src/lib/queries/categories.ts`
**Commit:** 26826c6
**Applied fix:** Changed `.set(data)` to `.set({ ...data, updatedAt: new Date() })` in `updateCategory`, matching the pattern used by all other update queries in the codebase.

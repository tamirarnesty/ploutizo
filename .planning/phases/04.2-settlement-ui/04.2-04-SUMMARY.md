## Deliverables

### StatCardPlaceholder.tsx
Ghost/live stat card component. Deferred cards render `$—` + "Available in Phase 7.3". Live cards (e.g. CREDIT CARD OWED) render formatted value + caption. `isPlaceholder` derived from `value === undefined`.

### SettlementSummaryPane.tsx
Per-member outstanding balance card. Consumes `useGetSettlements()` (shared `['settlements']` queryKey with Plan 05's CardBalancesGrid — TanStack Query dedup). Rolls up `balanceCents` and card count per member across all accounts. Negative balances render `text-success` green. "owed to them" subline (gender-neutral). Uses `OrgMember.displayName` and `imageUrl`.

### dashboardSearch.ts
Zod schema for `/dashboard` URL search params (`from`, `to` as optional strings). `catch(undefined)` prevents invalid values from blocking routing. Phase 4.2 writes these but does not use them for data fetching (D-02).

### PeriodRangePicker.tsx
Range calendar popover with 4 preset shortcuts (This month / Last month / Last 30 days / Last 7 days) rendered to the left of the calendar (D-19). Uses `useSearch({ from: '/_layout/dashboard' })` for reading and `useNavigate()` (without `from`) for writing. Writes `?from=YYYY-MM-DD&to=YYYY-MM-DD` to URL on selection; does not trigger data refetch.

### Dashboard.tsx
Page-level shell wiring all components. All queries fire at top level (no waterfalls). 4 stat cards in 2-col/4-col responsive grid. Two-column main row: `data-cardbalancesgrid-slot` placeholder div (left, for Plan 05) + SettlementSummaryPane (right, `w-72`). Month label from current date.

### _layout.dashboard.tsx
Route updated with `validateSearch: (search) => dashboardSearchSchema.parse(search)`.

### Auto-fixed Issues

**1. [Rule 1 - Bug] OrgMember field names corrected**
- **Found during:** Task 1
- **Issue:** Plan's `SettlementSummaryPane.tsx` code used `m.name` and `m.avatarUrl` from `useGetOrgMembers()`. The actual `OrgMember` type has `displayName` and `imageUrl`.
- **Fix:** Changed to `m.displayName` and `m.imageUrl ?? null`
- **Files modified:** `apps/web/src/components/dashboard/SettlementSummaryPane.tsx`
- **Commit:** c140223

**2. [Rule 1 - Bug] PeriodRangePicker useNavigate from constraint removed**
- **Found during:** Task 1 (pre-commit hook typecheck)
- **Issue:** `useNavigate({ from: '/_layout/dashboard' })` caused type error: `search` function return type not assignable to `never` due to TanStack Router search reducer type narrowing.
- **Fix:** Changed to `useNavigate()` without `from` option (Plan 03 executor applied this fix)
- **Files modified:** `apps/web/src/components/dashboard/PeriodRangePicker.tsx`
- **Commit:** c140223

**3. [Plan Defect] Acceptance criterion `grep -q "avatarUrl"` conflicts with correct implementation**
- **Issue:** Plan's acceptance criterion says `grep -q "avatarUrl"` should succeed with comment "(correct OrgMember field name, not imageUrl)". This is backwards — `avatarUrl` is the WRONG field name; the correct field is `imageUrl`. The criterion was authored against the buggy plan code.
- **Resolution:** Criterion fails as expected; `imageUrl` is the correct field and is present. Typecheck is the authoritative gate — it passes.

**4. [Prettier] StatCardPlaceholder className word order reordered**

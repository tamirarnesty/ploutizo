# Phase 4.2: Settlement UI - Context

**Gathered:** 2026-05-08
**Status:** Ready for planning

<domain>
## Phase Boundary

Build the Settlement UI on `/dashboard`: a credit card balances DataGrid with per-member breakdowns, a per-person settlement summary pane, and a lightweight Settle dialog. Also restructure sidebar nav (MEMBERS section + /expenses + /income routes + placeholder nav items). Period switcher is UI-only (not wired to data). `/settlements` full page added as placeholder route.

</domain>

<decisions>
## Implementation Decisions

### Dashboard Layout
- **D-01:** Dashboard layout = Settlement section + empty stat card placeholders. Phase 7.3 fills in charts/stats. No full layout scaffolding beyond this.
- **D-02:** Page includes a page header (e.g., "March 2026") + period selector. Period selector is UI-only in this phase — not wired to API data. TanStack Router search params for URL sync.

### Sidebar Nav Restructure
- **D-03:** Add a MEMBERS section to the sidebar, visible on every page. Each member shows name + avatar + a contextual value that changes per route (see D-10).
- **D-04:** Add placeholder nav items: Budget, Net Worth, Savings — rendered with a "soon" badge and disabled state. Do not rename existing nav items (they have diverged from old mockups).
- **D-05:** Split /transactions into dedicated filtered pages: add `/expenses` and `/income` routes now. Each page shows filtered transactions.
- **D-06:** Keep `/transactions` route as "All transactions" (do not redirect or rename).
- **D-07:** Add `/settlements` as a dedicated full page route (placeholder for now — Phase 7.3 builds it out).

### Settle CTA Flow
- **D-08:** Settle CTA style = lightweight shadcn Dialog (not a full page or side sheet).
- **D-09:** Dialog fields: Amount (pre-filled with member's outstanding balance), Source account (bank account select), Date, Notes (optional textarea).
- **D-10:** Settle CTA placement = per-member row inside the credit card breakdown, shown on hover only to keep the table minimal.
- **D-11:** Member selection in the dialog: show all household members with their balances; pre-select the first member with balance > 0. User can switch to any member before recording.
- **D-12:** Negative balance (member owes less than 0 / has a credit): display in green. Settle CTA remains visible — user may intentionally overpay; credits accumulate for future transactions.

### eachPersonPaysOwn Handling
- **D-13:** No special treatment in the UI. `eachPersonPaysOwn` was removed in Phase 4.1 (D-07 of 04.1-CONTEXT.md). All credit card accounts are treated uniformly. No split-card UI needed.

### Settlement Summary Pane
- **D-14:** Include a per-person settlement summary pane alongside the credit card table. Design: simple stat rows in a card — each row shows member name + outstanding balance (green if negative/credit).

### MEMBERS Sidebar Values (Route-Driven)
- **D-15:** Values injected into the MEMBERS sidebar section are derived client-side from each page's own query — no new API endpoints in Phase 4.2.
  - `/expenses` → member expense totals
  - `/income` → member income totals
  - `/dashboard` → member settlement balance
  - All other routes → blank/no value shown
- **D-16:** Full income-by-person data (GET /api/dashboard/income-by-person) is a Phase 7.3 concern. Phase 4.2 aggregates from existing transaction queries.

### Threshold / Notification Settings
- **D-17:** Settlement threshold settings live under Settings > Profile (new section). Out of scope for Phase 4.2 implementation — noted for future.

### UI Components
- **D-18:** Credit card balances table = ReUI DataGrid using native DataGrid features. Do NOT build a custom table from scratch. Mockup is inspiration only.
- **D-19:** Period selector = existing shadcn `Calendar` component from `packages/ui/src/components/calendar.tsx` with the Presets feature (react-day-picker). Reuse as-is.
- **D-20:** Settle dialog = shadcn Dialog primitive.
- **D-21:** Settlement summary pane = simple stat rows inside a card component (no DataGrid for this pane).

### Claude's Discretion
- Exact stat card placeholder content (ghost/skeleton styling vs. "coming soon" labels)
- Specific Tailwind token choices for layout spacing (follow existing `p-6` main padding from `_layout.tsx`)
- DataGrid column configuration details for the credit card breakdown (avatar + progress bar + amount + "owes X" text + hover Settle button)

</decisions>

- `/expenses` and `/income` routes filter the existing transactions API by type; no new endpoints

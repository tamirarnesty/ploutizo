# Plan 01: Build Text typography primitive

## Objective

Create `packages/ui/src/components/text.tsx` — the single typography primitive for the app. Implements a polymorphic `<Text as variant className>` component with decoupled `as` (DOM element) and `variant` (visual preset) props.

  h3: "font-heading text-xl font-semibold leading-snug",

**Component signature (React 19 — ref as regular prop, no forwardRef):**
```ts
type TextProps<T extends TextAs = "p"> = {
  as?: T
  variant?: TextVariant
  className?: string
  ref?: React.Ref<HTMLElement>
  children?: React.ReactNode
} & Omit<React.ComponentPropsWithoutRef<T>, "as" | "variant" | "className" | "ref" | "children">
```

**Component implementation:**
```tsx
function Text<T extends TextAs = "p">({
  as,
  variant = "body",
  className,
  ref,
  ...props
}: TextProps<T>) {
  const Tag = (as ?? "p") as React.ElementType
  return (
    <Tag
      ref={ref}
      className={cn(variantClasses[variant], className)}
      {...props}
    />
  )
}
```

**Exports at bottom of file:**
```ts
export { Text }
```

Full file structure:
1. `import * as React from "react"` 
2. `import { cn } from "@ploutizo/ui/lib/utils"`
3. Type exports: `TextAs`, `TextVariant`
4. `variantClasses` const
5. `TextProps` type (not exported — internal)
6. `Text` function component
7. `export { Text }`

No "use client" directive — this is a component library, consumers decide.

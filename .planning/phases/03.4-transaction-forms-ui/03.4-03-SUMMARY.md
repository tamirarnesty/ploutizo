## What Was Built

### AssigneeRow.tsx
Controlled single-row component with:
- `Avatar size="sm"` + `AvatarFallback` with initials via `getInitials` from TransactionColumns
- `Text as="span" variant="body-sm"` with `min-w-0 flex-1 truncate` for name overflow safety
- Mode-aware number input (`autoComplete="off"`, `w-20 text-right`):
  - Percent mode: patches both `percentage` and `amountCents` (Math.round(p/100*totalCents)); zero-guard not needed here
  - Dollar mode: patches both `amountCents` and `percentage` with zero-guard (`totalCents > 0 ? ... : 0`) preventing Infinity/NaN
- Display-only computed value in the opposite unit
- Ghost icon-button remove (`h-8 w-8`, 32px tap target) with `aria-label="Remove {memberName}"`

### SplitSection.tsx
Controlled section component with:
- Local `mode` state (`'percent' | 'dollar'`) only — all assignee data flows through `value`/`onChange` props
- `ToggleGroup` with `value={[mode]}` and `last-item guard` (prevents deselecting active toggle)
- Empty state copy exactly: "No assignees yet. Add one to split this transaction."
- `handleAddAssignee`: calls `lrmSplit(amountCents, [...existingIds, newId])` → even redistribution
- `handleRemove`: calls `lrmSplit(amountCents, remainingIds)` or `[]` if last member
- `handleRowChange`: patches individual row without LRM reset (for manual edits)
- Add-assignee `Select` filtered to exclude already-added members; disabled when `availableMembers.length === 0`
- Totals row: percentage sum + formatted total amount; mismatch warning when `totalPct !== 100%` and `value.length > 0`
- All handlers wrapped in `useCallback` for stable references (vercel-react-best-practices)

## Deviations from Plan

### Auto-fixed Issues

None — plan executed exactly as written with two minor implementation choices documented below.

### Implementation Choices (Claude's Discretion)

**1. Select value sentinel: `null` instead of `""`**
- Plan spec used `value=""` but base-ui `SelectRoot` types `value` as `Value | null` for non-multiple.
- Used `value={null}` as the "no selection" sentinel to satisfy TypeScript types correctly.

**2. Disabled state for fully-assigned picker**
- Plan did not specify behavior when all org members are already assigned.
- Added `disabled={availableMembers.length === 0}` on the Select trigger to prevent opening an empty popup.
- Covered by deviation Rule 2 (missing critical UX/correctness behavior).

**3. Floating-point mismatch check**
- Used `Math.round(totalPct * 10) !== 1000` instead of `totalPct !== 100` to avoid false positives from floating-point accumulation (e.g., 33.333 + 33.333 + 33.333 = 99.999).

# Phase 03.4 Plan 08: TransactionTagPicker Static JSX Fix Summary

**Fixed Base UI Combobox so existing tags appear in the dropdown and inline tag creation works — replaced silent render-function children and invalid root props with static JSX item rendering**

## Performance

- **Duration:** ~8 min

- **Completed:** 2026-04-17T00:08:00Z

- **Files modified:** 1

## Accomplishments
- Removed `items` and `filteredItems` props from `<Combobox>` root — these are not valid `ComboboxPrimitive.Root` props and were silently ignored, causing no items to render
- Replaced render-function children pattern `{(item: string) => <ComboboxItem>}` inside `<ComboboxList>` with static JSX `activeTags.map((tag) => <ComboboxItem>)` — ComboboxPrimitive.List is a plain `<ul>` and does not invoke children as a function
- Inline create option rendered as a static `<ComboboxItem key="__create__">` alongside tag items; Base UI's built-in filter keeps it visible because the user's input is a substring of the `__create__${tagInputValue}` value

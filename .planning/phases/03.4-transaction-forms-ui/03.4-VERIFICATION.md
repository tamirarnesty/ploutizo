## Human Verification Required (Updated)

The 7 UAT defects (plans 12-14 targets) are now closed by code. The original human verification flows remain — these now have updated expectations reflecting the fixed behavior.

#### 1. Create expense transaction

**Test:** Open Transactions page, click "Add transaction", select Expense type, fill Amount ($50.00), Date (via calendar popover), Account, Category, Description, submit.
**Expected:** Sheet closes, success toast ("Transaction created."), new transaction row appears in table.
**Why human:** Mutation round-trip, toast notification, table re-render, and sheet close sequence require live browser observation.

#### 2. Create refund with linked-transaction search

**Test:** Add transaction, select Refund type. Observe the "Refund of" dropdown on open (before typing).
**Expected:** Dropdown shows up to 10 recent expense transactions without typing. Then type 3 characters — dropdown switches to search results after ~300ms debounce.
**Why human:** Pre-load timing and combobox dropdown rendering require live interaction.

#### 3. Edit existing transaction

**Test:** Click ⋮ on a table row (button should be invisible at rest, appear on row hover). Choose "Edit".
**Expected:** Row hover reveals the ⋮ button. Sheet opens pre-populated. Amount shows formatted value (e.g. "1,000.00"). Clicking a calendar date closes the calendar automatically.
**Why human:** Row-hover CSS visibility, formatted amount rendering, and calendar auto-close are runtime behaviors.

#### 4. Delete transaction via icon-only button

**Test:** Open edit sheet for any transaction, click the trash icon button. Confirm in the AlertDialog.
**Expected:** AlertDialog opens. Clicking "Delete transaction" removes the transaction, closes the AlertDialog, closes the sheet, and shows "Transaction deleted." toast.
**Why human:** AlertDialog controlled close, post-delete onClose(), and toast require runtime observation.

#### 5. Switch between transactions (key-based remount)

**Test:** Edit transaction TX-A (fill or observe values), close sheet. Immediately edit transaction TX-B.
**Expected:** Form shows only TX-B's data. No TX-A values visible anywhere.
**Why human:** key={resolvedTransaction?.id ?? 'new'} remount behavior must be verified visually.

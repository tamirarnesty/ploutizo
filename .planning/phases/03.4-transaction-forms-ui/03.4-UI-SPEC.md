## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label hints, `gap-1` between icon+label |
| sm | 8px | Compact element spacing, `gap-2` within field rows |
| md | 16px | Field group vertical rhythm, horizontal Sheet padding (`px-6` = 24px → see lg) |
| lg | 24px | Sheet horizontal padding (`px-6`), section padding |
| xl | 32px | Gap between major form sections (base fields / type fields / split section) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- **Sheet width: 560px** — locked layout dimension (D-01); not on the 8-point scale. Override via `w-[560px] sm:w-[560px]` on `SheetContent`.
- **Sheet footer padding: `px-6 py-4`** — matches AccountForm footer exactly; inherit unchanged.
- **Touch targets for remove assignee button**: minimum 32px tap area (`h-8 w-8`) even if icon is 16px.

## Typography

Inherits the `Text` component variants established in Phase 03.3.2. The system uses Geist variable at the weights below.

| Role | Text Variant | Size | Weight | Line Height | Usage in this phase |
|------|-------------|------|--------|-------------|---------------------|
| Sheet title | `h3` (via `SheetTitle`) | 16px / 1rem | 500 | 1.2 | "Add transaction" / "Edit transaction" header |
| Body / input value | `body` | 14px / 0.875rem | 400 | 1.5 | Field inputs, Select values, Combobox results |
| Label | `label` | 14px / 0.875rem | 500 | 1.2 | `FieldLabel` for all form fields |
| Small / hint | `body-sm` | 12px / 0.75rem | 400 | 1.4 | "(optional)" label hint, import caption (└ Original: …) |
| Caption | `caption` | 12px / 0.75rem | 400 | 1.4 | Muted helper text, split section totals row |

Weights: 400 (body, hint, caption) and 500 (label, Sheet title). Size difference (16px vs 14px) provides hierarchy for the Sheet title without a third weight.

No new type sizes. Use `Text` component with `variant` prop only. Override color via `className` (e.g. `text-muted-foreground`).

## Color

Tokens reference `packages/ui/src/styles/globals.css`. Both light and dark modes are supported via CSS custom properties.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` | Page background behind open Sheet |
| Secondary (30%) | `--card` / `--muted` | Sheet surface (`SheetContent`), input backgrounds, Split section rows |
| Accent (10%) | `--primary` | Reserved — see below |
| Destructive | `--destructive` | Delete button text/icon in footer; AlertDialog "Delete transaction" action |
| Muted text | `--muted-foreground` | "(optional)" hints, import caption, placeholder text, caption text |
| Border | `--border` | Sheet header border, footer border, field input borders, split row separators |
| Success | `--success` | Not used in this phase |

**Accent (`--primary`) reserved for:**
- Primary submit Button (`variant="default"`) — "Add transaction" and "Save changes"
- Active/focused field ring (`ring-primary`) — visible focus state on inputs and Selects
- Selected item in Select trigger (inherited from shadcn SelectItem)
- "Add assignee" Button when in active/hover state

Accent is NOT used for: section headings, labels, disabled states, or informational text.

**Destructive (`--destructive`) reserved for:**
- "Delete transaction" trigger text/icon in edit form footer
- AlertDialog destructive action button ("Delete transaction")
- Field validation error text via `FieldError` component

## Interaction Contracts

### TransactionSheet
- Opens from: "Add transaction" button (create) and "Edit" action in table ⋮ dropdown (edit)
- Width: 560px, anchored right
- Overlay: Sheet backdrop dims the transaction list (stays visible behind the Sheet)
- Close on: Sheet close button, `Escape`, "Discard changes" button, successful submit, successful delete
- Title: "Add transaction" (create) | "Edit transaction" (edit) — `SheetTitle`
- `key` prop at call site: `key={transaction?.id ?? "new"}` — forces remount on transaction switch

### Type Select (D-06)
- Standard `Select` component, always enabled in both create and edit modes
- On value change: clear type-specific fields of previous type; preserve shared fields (D-07)
- Field ordering: type Select is the first field in the form, before accountId/amount/date

### Type-Specific Fields (D-08 to conditional per type)
- Rendered via `form.Subscribe` keyed on `type` field value
- `expense`: categoryId Select — required
- `refund`: categoryId Select — optional; refundOf Combobox — optional (Refund Linker)
- `income`: incomeType Select — required; incomeSource Input — optional
- `transfer`: toAccountId Select — required (distinct from fromAccountId/accountId)
- `settlement`: settledAccountId Select — optional
- `contribution`: investmentType Select — optional
- Required fields use standard `FieldError` on blur + submit; optional fields show "(optional)" hint in `FieldLabel`

### Refund Linker Combobox (D-14 to D-17)
- Label: "Refund of" with `(optional)` hint inline in `FieldLabel`
- Popover opens on trigger click; `CommandInput` is auto-focused inside popover
- Query fires on `input.length >= 2`, debounced 300ms
- Each result row: `{description || merchant} • {date formatted as MMM D, YYYY} • {formatCurrency(amountCents)}`
- Selecting a result: pre-fills split section with original transaction's assignees (D-16)
- Clearing the field: does NOT reset split (D-17)
- Empty state in list: "No matching transactions found."
- Loading state in list: `Spinner` centered in `CommandList`

### Split Section (D-09 to D-13)
- Always visible below type-specific fields — no collapse
- Header row: "Split" label (`Text variant="label"`) + % | $ `ToggleGroup` toggle (right-aligned)
- Empty state: `Text variant="body-sm" className="text-muted-foreground"` — "No assignees yet. Add one to split this transaction."
- "Add assignee" button: below assignee list; opens member picker (Select or Combobox per org size)
- Assignee row: Avatar + member name (`min-w-0` on name span) + editable input + computed display + remove button
- Remove button: icon-only (`X` from lucide, 16px), `aria-label="Remove {memberName}"`, min 32px tap area
- On add/remove assignee: reset all rows to even split via LRM
- Totals row: shows sum of % column and sum of $ column; muted text; shows warning if % ≠ 100% or $ ≠ amount
- % mode: % input editable, $ computed from percentage × amount
- $ mode: $ input editable, % computed from amount_cents / total_cents × 100

### Import Caption (D-19 — conditional)
- Shown only when `originalDescription` / `originalMerchant` exists AND differs from current value
- Renders directly below the relevant Input as a `Text variant="caption" className="text-muted-foreground"` with prefix: `└ Original: {value}`
- If schema columns are absent, skip rendering; note as deferred (D-19 deferred flag)

### Form Footer
- `border-t border-border px-6 py-4` — matches AccountForm footer
- Left slot: "Delete transaction" ghost button (edit mode only) — `Trash2` icon (16px) + "Delete transaction" text; `className="text-destructive hover:text-destructive"`
- Right slot: "Discard changes" outline Button + primary submit Button ("Add transaction" | "Save changes")
- Submit button shows `Spinner` when `isSubmitting === true` and is `disabled`

### Delete Confirm Dialog (D-05)
- Trigger: "Delete transaction" button in edit form footer
- `AlertDialog` (not `Dialog`) — blocks interaction
- Title: "Delete transaction?"
- Body: "This will permanently remove this transaction. This action cannot be undone."
- Cancel: "Cancel"
- Action: "Delete transaction" — `variant="destructive"`
- On confirm: `useDeleteTransaction.mutate(id)` → on success: `onClose()` + invalidate `["transactions"]`

### Loading Gate
- While `useGetTransaction` (edit) or `useGetOrgMembers` is loading: render `Spinner` centered in `flex-1` area (matches AccountForm loading gate pattern)
- Form mounts only when all required data is loaded — no `useEffect`/`form.reset`

### Focus Management
- On Sheet open: `autoFocus` on the Type Select — it is the primary purpose of the user action (justification: user opens sheet to record a transaction; type is the first decision)
- All inputs have `autocomplete` attributes:
  - description: `autocomplete="off"`
  - merchant: `autocomplete="off"`
  - amount: `autocomplete="off"`
  - incomeSource: `autocomplete="off"`
  - All other fields: `autocomplete="off"` (no standard semantic values apply to financial classification fields)

| Element | Copy |
|---------|------|
| Sheet title — create | Add transaction |
| Sheet title — edit | Edit transaction |
| Primary CTA — create | Add transaction |
| Primary CTA — edit | Save changes |
| Discard button | Discard changes |
| Delete trigger (footer) | Delete transaction |
| Empty split heading | (none — inline body only) |
| Empty split body | No assignees yet. Add one to split this transaction. |
| Add assignee button | Add assignee |
| Refund of label | Refund of |
| Refund of hint | (optional) |
| Refund combobox placeholder | Search transactions… |
| Refund combobox empty | No matching transactions found. |
| Import caption prefix | └ Original: |
| Form-level mutation error | Couldn't save changes. Check your connection and try again. |
| Delete dialog title | Delete transaction? |
| Delete dialog body | This will permanently remove this transaction. This action cannot be undone. |
| Delete dialog cancel | Cancel |
| Delete dialog action | Delete transaction |
| Split totals mismatch warning | Percentages must add up to 100%. |
| Field required error (generic) | This field is required. |
| Type-specific: categoryId required | Category is required for expense transactions. |
| Type-specific: incomeType required | Income type is required. |
| Type-specific: toAccountId required | Transfer destination account is required. |

Copy rules:
- Use `…` (U+2026) not `...` for ellipsis (e.g., "Search transactions…")
- Use sentence case for all labels and error messages
- Never truncate transaction names in the refund combobox without `min-w-0` + `truncate`

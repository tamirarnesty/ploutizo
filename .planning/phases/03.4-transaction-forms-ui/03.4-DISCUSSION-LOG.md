## Create vs Edit — Type Mutability

| Option | Description | Selected |
|--------|-------------|----------|
| Fully mutable (Select in both modes) | Type changeable in edit; PATCH uses discriminated union | ✓ |
| Immutable in edit (read-only badge) | Keep updateTransactionSchema as-is | |

**User's choice:** Fully mutable
**User's reasoning:** "If I accidentally choose expense but it was actually a refund, I should be able to modify it." No technical reason to lock type post-creation (all type-specific columns are nullable in the DB).
**Notes:** API impact — updateTransactionSchema must accept type as discriminant.

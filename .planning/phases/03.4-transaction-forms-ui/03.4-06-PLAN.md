<interfaces>
<!-- Base UI Select.Value does NOT auto-resolve labels from SelectItem children. -->
<!-- Items live in a Portal that only mounts when the popover is open. -->
<!-- Fix: pass explicit children to SelectValue that compute the label inline. -->

Pattern for enum selects:
```tsx
const TYPE_LABELS: Record<string, string> = {
  expense: 'Expense', income: 'Income', transfer: 'Transfer',
  settlement: 'Settlement', refund: 'Refund', contribution: 'Contribution',
}
// In JSX:
<SelectValue>{TYPE_LABELS[field.state.value] ?? 'Select type'}</SelectValue>
```

Pattern for array-backed selects (accounts, categories):
```tsx
<SelectValue>
  {accounts.find((a) => a.id === field.state.value)?.name ?? 'Select account'}
</SelectValue>
```

When value is empty string (''), the lookup returns undefined, so the fallback
'Select account' text is shown — matching the former placeholder behavior.
</interfaces>
</context>

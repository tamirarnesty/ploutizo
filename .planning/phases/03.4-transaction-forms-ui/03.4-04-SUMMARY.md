## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Select onValueChange null guard**
- **Found during:** Task 2 (TypeScript compilation)
- **Issue:** TanStack Form's `handleChange` expects `Updater<string>`, but `Select.onValueChange` may pass `string | null`. TypeScript error on all 6 Select fields in TransactionTypeFields.
- **Fix:** Changed `onValueChange={(v) => field.handleChange(v)}` to `onValueChange={(v) => { if (v !== null) field.handleChange(v) }}` on all Select fields.
- **Files modified:** TransactionTypeFields.tsx, TransactionForm.tsx
- **Commit:** 5dc52d6 (fix applied inline before commit)

**2. [Rule 2 - Design] Tag picker items = names, not UUIDs**
- **Found during:** Task 3 pre-implementation review (advisor)
- **Issue:** Plan specified `items = activeTags.map(t => t.id)` (UUIDs as items). base-ui Combobox filters on item values — UUID items would make text search non-functional.
- **Fix:** Items are tag names. `onValueChange` translates selected names → UUIDs before updating form field. Form field still contains only real UUIDs. `filteredItems` prop used for `__create__` option (bypasses base-ui filtering).
- **Files modified:** TransactionTagPicker.tsx
- **Commit:** 59d6a5a

## Key Decisions Made

1. **Tag items as names**: base-ui Combobox has no custom filter prop — items must be readable strings for search to work. Translation to UUIDs happens in `handleValueChange`.
2. **Null guard on Select.onValueChange**: All Select fields use `if (v !== null)` guard instead of type cast to satisfy TanStack Form's `Updater<string>` type.

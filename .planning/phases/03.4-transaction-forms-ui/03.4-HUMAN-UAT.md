## Current Test

Human testing completed 2026-04-18. 7 defects found across 4 test flows.

## Tests

### 1. Create expense transaction
expected: Sheet closes, success toast, new transaction row appears in table with correct values
result: failed — no toast shown; transaction appears to create (row in table) but no user feedback

### 2. Create refund with linked-transaction search
expected: Dropdown appears with matching transactions after debounce; select one; form submits successfully
result: failed — (a) no items pre-loaded on open; (b) search returns no results for valid input (same items/render-fn bug as TagPicker)

### 3. Edit existing transaction
expected: Sheet opens pre-populated — amount in dollars, date as formatted string, tags as chips, type sub-form fields filled
result: partial — prefill works; amount shows raw number without comma formatting (1000 not 1,000); calendar stays open after date selection; edit button always visible (should be row-hover only)

### 4. Delete transaction via icon-only button
expected: AlertDialog opens on click, transaction removed after confirm, sheet closes
result: failed — (a) AlertDialog does not close after confirm (AlertDialogAction is a plain Button, not Close primitive); (b) Sheet does not close; (c) delete API not confirmed to succeed; (d) stale data shown even after page refresh

### 5. Switch between transactions (key-based remount)
expected: Form shows only TX-B's data after switching from TX-A — no stale values
result: passed — key remount works correctly

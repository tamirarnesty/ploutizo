Create `apps/web/src/components/transactions/types.ts`.

This file defines the flat superset form interface and the assignee row type consumed by SplitSection, AssigneeRow, and useTransactionForm.

**Why flat superset (not discriminated union):** TanStack Form's `name=` prop must be a keyof the form values type. A discriminated union would require type narrowing on every `form.AppField` call, which is not supported by TanStack Form v1. All type-specific fields are optional in the interface — the schema validator (run in onSubmit via toApiPayload) enforces type-specific requirements server-side.

**AssigneeFormRow:** Stores percentage as `number` (not string). The Drizzle `transactionAssignees.percentage` column returns `string | null` from the DB driver — always `parseFloat()` when reading from a GET response before populating form state.

**tagIds:** Stores an array of tag UUIDs. The form field value contains only IDs — tag names are resolved from the `useGetTags` response for display. The schema field name is `tagIds` (confirmed in createTransactionSchema). The `TransactionRow.tags` array contains `{ id, name, colour }` objects — map to `tagIds` by extracting `.id`.

```ts
/**
 * Flat superset of all 6 transaction type field shapes.
 *
 * Why flat (not discriminated union): TanStack Form name= prop requires keyof
 * to be statically known. A discriminated union would require runtime type narrowing
 * on every form.AppField call, which TanStack Form v1 does not support.
 *
 * All type-specific fields are optional. Server-side validation (createTransactionSchema)
 * enforces type-specific required fields after toApiPayload transforms the form values.
 */
export interface TransactionFormValues {
  // Shared base fields (preserved across type changes — D-07)
  type: 'expense' | 'income' | 'transfer' | 'settlement' | 'refund' | 'contribution'
  accountId: string
  amount: number         // dollars (form); API/DB stores cents — see toApiPayload
  date: string           // ISO date string 'YYYY-MM-DD'
  description: string
  merchant: string
  tagIds: string[]       // UUIDs of selected tags; names resolved via useGetTags for display

  // expense + refund (optional for refund)
  categoryId: string

  // refund
  refundOf: string       // UUID of the original transaction; empty string = none

  // income
  incomeType: string     // 'direct_deposit' | 'e_transfer' | 'cash' | 'cheque' | 'other'
  incomeSource: string

  // transfer
  toAccountId: string

  // settlement
  settledAccountId: string

  // contribution
  investmentType: string // 'tfsa' | 'rrsp' | 'fhsa' | 'resp' | 'non_registered' | 'other'

  // Split section
  assignees: AssigneeFormRow[]
}

/**
 * One row in the SplitSection assignee list.
 *
 * percentage is stored as a number here (not string).
 * When reading from a GET response, always parseFloat(row.percentage ?? '0')
 * before populating form state — Drizzle returns numeric columns as strings.
 */
export interface AssigneeFormRow {
  memberId: string
  amountCents: number
  percentage: number
}
```

**useGetTransaction.ts:**

Fetches a single transaction by ID. Used by TransactionForm in edit mode.

- `enabled: id !== null` — never fires when id is null (create mode)
- queryKey: `['transactions', id]` — nested under the same 'transactions' key so broad invalidation from create/update/delete clears it automatically
- Returns `TransactionRow` (same joined shape as the list response — includes assignees and tags)
- Re-export `TransactionRow` and `TransactionAssignee` from useGetTransactions so downstream imports use a single path

```ts
import { useQuery } from '@tanstack/react-query'
import type { UseQueryResult } from '@tanstack/react-query'
import { apiFetch } from '@/lib/queryClient'
import type { TransactionRow } from './useGetTransactions'

export const fetchTransaction = async (id: string): Promise<TransactionRow> => {
  const r = await apiFetch<{ data: TransactionRow }>(`/api/transactions/${id}`)
  return r.data
}

export const useGetTransaction = (id: string | null): UseQueryResult<TransactionRow> => {
  return useQuery({
    queryKey: ['transactions', id],
    queryFn: () => fetchTransaction(id!),
    enabled: id !== null,
  })
}
```

**useSearchTransactions.ts:**

Searches transactions by description substring for the Refund Linker combobox. Debouncing is handled at the component level via a debounced state value passed in — this hook fires only when the description is long enough.

- `enabled: description.length >= 2` — no API call for empty or single-character input
- queryKey: `['transactions', 'search', description]` — separate from the list cache
- Returns `TransactionRow[]` (uses the list endpoint with description + limit=20)
- The 300ms debounce is applied by the caller (RefundLinker component) before passing description into this hook — do NOT add a setTimeout or useEffect debounce inside this hook

```ts
import { useQuery } from '@tanstack/react-query'
import type { UseQueryResult } from '@tanstack/react-query'
import { apiFetch } from '@/lib/queryClient'
import type { TransactionRow, TransactionListResponse } from './useGetTransactions'

export const fetchSearchTransactions = async (description: string): Promise<TransactionRow[]> => {
  const qs = new URLSearchParams()
  qs.set('description', description)
  qs.set('limit', '20')
  const r = await apiFetch<TransactionListResponse>(`/api/transactions?${qs.toString()}`)
  return r.data
}

export const useSearchTransactions = (description: string): UseQueryResult<TransactionRow[]> => {
  return useQuery({
    queryKey: ['transactions', 'search', description],
    queryFn: () => fetchSearchTransactions(description),
    enabled: description.length >= 2,
  })
}
```

**useCreateTransaction.ts:**

The mutationFn accepts the full API payload (already transformed to cents by toApiPayload in useTransactionForm — this hook does NOT convert dollars to cents). The response returns the created TransactionRow.

```ts
import { useMutation, useQueryClient } from '@tanstack/react-query'
import { apiFetch } from '@/lib/queryClient'
import type { TransactionRow } from './useGetTransactions'

export const useCreateTransaction = () => {
  const qc = useQueryClient()
  return useMutation({
    mutationFn: (body: unknown) =>
      apiFetch<{ data: TransactionRow }>('/api/transactions', {
        method: 'POST',
        body: JSON.stringify(body),
      }).then((r) => r.data),
    onSettled: () => qc.invalidateQueries({ queryKey: ['transactions'] }),
  })
}
```

Note: `body: unknown` is intentional — the payload is produced by `toApiPayload` in `useTransactionForm`, which validates via createTransactionSchema.safeParse before calling mutate. This hook is a thin transport layer and does not re-validate.

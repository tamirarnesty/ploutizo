# Phase 03.4 Plan 15: Delete/Fetch UX Fixes Summary

## What Was Built

Deferred the optimistic list cache update in `useDeleteTransaction` from `onMutate` to `onSuccess`, so the deleted row disappears from the table only after the toast and sheet close have fired. Also added `initialData` support to `useGetTransaction` and wired it into `TransactionForm` so opening the edit sheet with a row already in the table cache never shows the fetch spinner.

## Tasks Completed

1. **Moved setQueriesData to onSuccess in useDeleteTransaction** — `onMutate` now only calls `cancelQueries` and takes a snapshot. `onSuccess` filters the deleted row from all list caches. `onError` rollback and `onSettled` invalidation are unchanged.
2. **Added initialData support to useGetTransaction + wired in TransactionForm** — Hook signature extended with `options?: { initialData?: TransactionRow }`, passed through to `useQuery`. `TransactionForm` passes `{ initialData: transaction ?? undefined }` in edit mode.

## Key Files

- `apps/web/src/lib/data-access/transactions/useDeleteTransaction.ts` — onMutate slimmed; onSuccess added with setQueriesData
- `apps/web/src/lib/data-access/transactions/useGetTransaction.ts` — optional initialData param
- `apps/web/src/components/transactions/TransactionForm.tsx` — initialData wired from table row

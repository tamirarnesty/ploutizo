<objective>
Fix the row action button to be invisible at rest and visible only when the containing table row is hovered.

The previous gap plan (03.4-09) made the button always visible (`text-muted-foreground hover:text-foreground`), but the user wants it to appear only on row hover. The `group-hover:opacity-100` pattern from the original code failed because DataGrid `<tr>` elements have no `group` class.

Fix: Use Tailwind v4's arbitrary parent-selector variant `[tr:hover_&]:opacity-100` which applies when any ancestor `<tr>` is hovered. This works without adding a `group` class to the `<tr>`.
</objective>

<context>
<interfaces>
<!-- Tailwind v4 arbitrary parent selector syntax -->
<!-- [tr:hover_&] means: when an ancestor <tr> is hovered, apply to this element (&) -->
<!-- The underscore encodes a space (descendant combinator in CSS) -->

Target className on the DropdownMenuTrigger's render Button:
```tsx
className="opacity-0 [tr:hover_&]:opacity-100 data-[state=open]:opacity-100 focus-visible:opacity-100"
```

This replaces the current `"text-muted-foreground hover:text-foreground"` from plan 09.

Why this works: DataGrid renders rows as `<tr>` elements. When the user hovers anywhere on the row (any `<td>`), the `<tr>:hover` CSS state fires. The `[tr:hover_&]` arbitrary variant generates:
  `tr:hover .className { opacity: 1 }`
which correctly reveals the button regardless of which cell is hovered.
</interfaces>
</context>

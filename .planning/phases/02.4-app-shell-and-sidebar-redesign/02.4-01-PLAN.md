**Step B — Rewrite `apps/web/src/routes/_layout.tsx`** (per D-04, D-05, D-06):

CRITICAL structural note from RESEARCH.md Pattern 1 + Pitfall 1:
- `SidebarTrigger` (in TopBar) calls `useSidebar()` which requires `SidebarProvider` context.
- Therefore `SidebarProvider` MUST wrap `TopBar` in the component tree, even though TopBar visually spans above the sidebar.
- The solution: put `SidebarProvider` at the outermost level; use an inner `flex flex-col h-screen w-full` div to achieve the visual top-bar-above-content layout.

```tsx
import { Outlet, createFileRoute } from "@tanstack/react-router"
import {
  SidebarInset,
  SidebarProvider,
} from "@ploutizo/ui/components/sidebar"
import { AppSidebar } from "../components/AppSidebar"
import { TopBar } from "../components/TopBar"

export const Route = createFileRoute("/_layout")({
  component: LayoutShell,
})

function LayoutShell() {
  return (
    <SidebarProvider>
      <div className="flex h-screen w-full flex-col">
        <TopBar />
        <div className="flex flex-1 min-h-0">
          <AppSidebar />
          <SidebarInset>
            <main className="flex-1 overflow-auto p-6">
              <Outlet />
            </main>
          </SidebarInset>
        </div>
      </div>
    </SidebarProvider>
  )
}
```

Key points:
- `SidebarTrigger` import removed from `_layout.tsx` (it now lives in TopBar and AppSidebar)
- The `h-14` header inside `SidebarInset` is removed entirely (per D-04)
- `TopBar` is a separate named component imported from `../components/TopBar` — not defined inline (per CLAUDE.md `rerender-no-inline-components`)
- `SidebarProvider` wraps everything so `TopBar`'s `SidebarTrigger` has access to `useSidebar()` context
- The inner `div className="flex flex-1 min-h-0"` holds the sidebar + inset row

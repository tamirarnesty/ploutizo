# Phase 02.4 Plan 02: Settings Sub-Nav Layout Summary

**One-liner:** Two-column SettingsLayout with w-48 vertical sub-nav (Categories & Tags, Merchant Rules, Household) + flex-1 Outlet using TanStack Router active state detection.

## What Was Built

Rewrote `apps/web/src/routes/_layout.settings/route.tsx` from a pass-through `<Outlet />` to a two-column layout component with a vertical navigation bar on the left and route content on the right.

### Task 1: Add two-column SettingsLayout to _layout.settings/route.tsx

**Commit:** `24d0e10`
**Files:** `apps/web/src/routes/_layout.settings/route.tsx`

The `SettingsLayout` component now renders:
- Left column: `<nav aria-label="Settings navigation">` with `w-48 shrink-0`, containing 3 `<Link>` items (Categories & Tags, Merchant Rules, Household) with Tag, Store, Home icons
- Active detection via `useRouterState()` using exact `location.pathname === to` match
- Active state: `bg-accent font-medium text-foreground`; inactive state: `text-muted-foreground hover:bg-accent hover:text-foreground`
- Right column: `<div className="min-w-0 flex-1">` containing `<Outlet />`
- Outer wrapper: `<div className="flex gap-8">` — no `p-6` (shell's `<main className="p-6">` handles padding)

## Decisions Made

1. **No double-padding:** The `SettingsLayout` wrapper uses `flex gap-8` without `p-6`. The outer `<main className="flex-1 overflow-auto p-6">` in `_layout.tsx` already pads all route content. Adding `p-6` again would produce 48px total padding, mismatching other pages. (RESEARCH.md Pitfall 3)

2. **Exact pathname match:** `location.pathname === to` (not `startsWith`) for sub-nav active state. This ensures only the exact settings sub-route is highlighted — navigating to `/settings/categories/some-child` would not accidentally highlight a wrong item.

3. **Module-scope `settingsNavItems`:** The nav items array is hoisted to module scope as `const` (not defined inside `SettingsLayout`). This follows `server-hoist-static-io` rule — static data should not be recreated on each render.

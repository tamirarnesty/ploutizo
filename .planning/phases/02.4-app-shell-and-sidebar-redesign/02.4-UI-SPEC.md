## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, icon-only rail icon centering |
| sm | 8px | Compact element spacing, sidebar footer button gap |
| md | 16px | Default element spacing, top bar horizontal padding |
| lg | 24px | Section padding, page content padding (existing `p-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Top bar height: 40px (slim bar — not a full h-14 content header). Use `h-10`.
- Sidebar header row height: 48px (`h-12`) to accommodate "Ploutizo" wordmark + `OrganizationSwitcher` side-by-side.
- Touch targets (mobile `SidebarTrigger` in top bar): 44px minimum hit area.

Source: CONTEXT.md D-04/D-05; existing `p-6` in `LayoutShell.tsx`

## Color

All values reference CSS custom properties from `packages/ui/src/styles/globals.css`. Never use raw hex or `oklch()` literals in component code — always use semantic tokens.

| Role | Token | Value (light) | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` = white | Page content area surface |
| Secondary (30%) | `--sidebar` | `oklch(0.985 0.001 106.423)` = near-white warm | Sidebar surface; top bar surface |
| Accent (10%) | `--sidebar-primary` | `oklch(0.588 0.158 241.966)` = blue | Active nav item indicator, active `SidebarMenuButton` highlight |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` = red | Destructive actions only (none in this phase) |

Accent reserved for:
- Active `SidebarMenuButton` background/indicator (via `isActive` prop — handled by shadcn Sidebar automatically)
- Active "Settings" footer link state

Top bar surface: `bg-background` with a `border-b border-border` — 1px bottom border only (no shadow). Height `h-10`. This is a structural divider, not a content header.

Dark mode: All tokens above have dark counterparts in globals.css; no additional overrides needed.

Source: CONTEXT.md (Claude's Discretion — top bar treatment); globals.css

## Layout Contract

This phase restructures the shell. The following layout decisions are binding for the executor.

### Shell Structure (after redesign)

```
<div class="flex flex-col h-screen">
  <TopBar />                          <!-- h-10, full width, bg-background, border-b -->
  <div class="flex flex-1 min-h-0">
    <AppSidebar />                    <!-- collapsible="icon" -->
    <SidebarInset>
      <main class="flex-1 overflow-auto p-6">
        <Outlet />
      </main>
    </SidebarInset>
  </div>
</div>
```

Note: `SidebarProvider` wraps the inner flex row (sidebar + inset), not the entire screen.
`TopBar` sits above `SidebarProvider` so it spans the full viewport width.

### Top Bar (`TopBar` component — new)

- Height: `h-10`
- Background: `bg-background`
- Bottom border: `border-b border-border`
- Layout: `flex items-center justify-between px-4`
- Left slot (mobile only): `<SidebarTrigger />` — hidden on desktop via `md:hidden`
- Right slot: `<UserButton />` from `@clerk/tanstack-react-start` — always visible
- No breadcrumbs, no page title (CONTEXT.md D-07)

### Sidebar Header (updated)

- Single row: `flex items-center justify-between px-3 h-12`
- Left: "Ploutizo" wordmark — `<span>` with `text-base font-semibold font-sans text-sidebar-foreground`
- Right: `<OrganizationSwitcher hidePersonal afterCreateOrganizationUrl="/dashboard" afterSelectOrganizationUrl="/dashboard" />`
- In icon-only collapse state: entire `<SidebarHeader>` collapses to nothing (CONTEXT.md D-09). Use `group-data-[collapsible=icon]:hidden` on the header content or rely on shadcn's built-in group-data classes on `<SidebarHeader>`.

### Sidebar Collapse

- `<Sidebar collapsible="icon">` — full width ↔ icon-only rail (CONTEXT.md D-01)
- No fully-hidden state on desktop
- Collapse toggle button: `<SidebarTrigger />` or a `<Button variant="ghost" size="icon">` with `<PanelLeft />` icon, placed in `<SidebarFooter>` right side (CONTEXT.md D-02)
- Icon: `PanelLeft` from `lucide-react`
- Footer layout: `flex items-center justify-between px-2 py-2`
  - Left: `<ThemeToggle />` (already present)
  - Right: collapse toggle button

### Primary Nav

- `navItems` array: Dashboard (`LayoutDashboard`), Accounts (`CreditCard`)
- Rendered as `<SidebarMenuButton asChild isActive={active}>` wrapping TanStack Router `<Link>`
- No group label for primary nav (flat list)

### Settings in Footer (updated)

- Remove `settingsItems` group from `<SidebarContent>`
- Add single "Settings" footer link in `<SidebarFooter>` left area (alongside ThemeToggle)
- Footer layout: `flex items-center gap-2 px-2 py-2`
  - Left cluster: `<ThemeToggle />` + Settings link
  - Right: collapse toggle button
- Settings link: `<SidebarMenuButton asChild isActive={isSettingsActive}>` wrapping `<Link to="/settings">` with `<Settings />` icon + "Settings" label

### Settings Sub-Nav Layout (`/settings` route)

- The `SettingsLayout` component (`_layout.settings/route.tsx`) gains a two-column layout:
  - Left: vertical sub-nav (`w-48 shrink-0`) with 3 items: Categories & Tags, Merchant Rules, Household
  - Right: `<Outlet />` content area (`flex-1 min-w-0`)
  - Container: `flex gap-8 p-6`
- Sub-nav items: `<nav>` containing `<Link>` items styled as vertical tab-style links
  - Active: `text-foreground font-medium bg-accent rounded-md`
  - Inactive: `text-muted-foreground hover:text-foreground hover:bg-accent rounded-md`
  - Padding per item: `px-3 py-2`
  - Icons alongside labels: same icons as before (Tag, Store, Home)
- The main app sidebar shows "Settings" footer link as active while any `/settings/*` route is active

## Interaction Contract

### Sidebar Collapse Toggle

- Clicking the collapse toggle (`PanelLeft` icon button in footer) calls the `useSidebar()` hook `toggleSidebar()` method — OR uses `<SidebarTrigger>` which does this automatically.
- Transition: shadcn Sidebar handles CSS transition on collapse; no custom animation needed.
- Collapsed state persists in `SidebarProvider`'s internal state (no server persistence required in v1).

### Nav Active States

- Primary nav: `active = location.pathname === to || location.pathname.startsWith(to + "/")` — existing pattern, unchanged.
- Settings footer link: `active = location.pathname.startsWith("/settings")`.
- In icon-only mode, `SidebarMenuButton` shows a `Tooltip` with the label on hover — this is handled automatically by shadcn Sidebar when the sidebar is collapsed.

### Mobile Behavior

- `<SidebarTrigger />` in top bar left slot (visible only on mobile via `md:hidden`).
- Sidebar opens as drawer overlay on mobile — unchanged from Phase 02 behavior.
- Top bar `<SidebarTrigger />` is `className="md:hidden"`.

### Settings Sub-Nav

- Active sub-nav item detected via `location.pathname === to` (exact match).
- Clicking a sub-nav item navigates to the sub-route; the main sidebar stays visible with "Settings" active.
- No accordion or expand/collapse — flat vertical list.

# Phase 02.4: app shell and sidebar redesign - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the existing app shell (`LayoutShell.tsx`) and sidebar (`AppSidebar.tsx`) built in Phase 02 into their final form — the layout and navigation structure that all future phases build on.

Scope:
- Sidebar collapse to icon-only rail (toggle button in sidebar footer)
- Replace bare h-14 header with a slim full-width top bar (UserButton only)
- Sidebar header row: "Ploutizo" wordmark + OrganizationSwitcher side-by-side
- Settings nav moved out of sidebar into a /settings page with vertical sub-nav tabs
- Settings becomes a bottom sidebar footer link

Out of scope:
- Search bar / ⌘K palette (future phase)
- Notification bell (future phase — wired to notifications system)
- Any new route pages beyond shell/layout changes

</domain>

<decisions>
## Implementation Decisions

### Sidebar Collapse
- **D-01:** Sidebar collapses to icon-only rail using shadcn Sidebar `collapsible="icon"`. Full width ↔ icon rail only — no fully-hidden state on desktop.
- **D-02:** Collapse toggle button lives at the bottom of the sidebar footer (right side of footer row), matching the mockup's split-view icon placement.
- **D-03:** In icon-only mode, the sidebar header (Ploutizo wordmark + OrganizationSwitcher) collapses cleanly to nothing — no "P" monogram or icon mark needed.

### App Header / Top Bar
- **D-04:** Remove the current `<header className="flex h-14 shrink-0 items-center gap-2 border-b px-4">` from `LayoutShell.tsx` entirely.
- **D-05:** Add a slim top bar that spans the full app width (above both sidebar and content area) — rendered at the app root level, not inside `SidebarInset`. Contains only `<UserButton />` pinned to the right.
- **D-06:** On mobile, the `<SidebarTrigger />` (hamburger) moves into this top bar on the left side — replaces its current position in the removed header.
- **D-07:** No breadcrumbs, no page title in the top bar. Each page owns its own heading.

### Sidebar Header (Branding + Org Switcher)
- **D-08:** Sidebar `<SidebarHeader>` contains a single row with two elements, vertically centered:
  - Left: "Ploutizo" text wordmark (typography only, no icon mark)
  - Right: `<OrganizationSwitcher />` component (existing props: `hidePersonal`, `afterCreateOrganizationUrl`, `afterSelectOrganizationUrl`)
- **D-09:** In icon-only collapsed state, the header row collapses away entirely — first visible element in the icon rail is the top nav item.

### Settings Placement
- **D-10:** The current `<SidebarGroup>` labeled "Settings" with 3 items (Categories & Tags, Merchant Rules, Household) is removed from the sidebar.
- **D-11:** A single "Settings" link (gear icon + label) is added to the sidebar footer area alongside Support (if added) — a flat bottom nav item, not a group.
- **D-12:** `/settings` renders a dedicated layout: vertical sub-nav tabs on the left listing the 3 sub-sections (Categories & Tags, Merchant Rules, Household). The existing `/settings/categories`, `/settings/merchant-rules`, `/settings/household` routes are unchanged — only the navigation to them changes.
- **D-13:** The settings sub-nav is inside the page content area (not the app sidebar). When navigating to settings, the main sidebar stays showing the primary nav (with "Settings" active).

### Mobile Behavior
- **D-14:** On mobile, sidebar still opens as a drawer overlay (unchanged from Phase 02 D-06). The `<SidebarTrigger />` moves to the top bar left side (see D-06).

### Claude's Discretion
- Exact styling of the "Ploutizo" wordmark (font weight, size, color — use existing CSS variables)
- Top bar height and shadow/border treatment
- Collapse toggle icon choice (PanelLeft or similar from lucide-react)
- Active state styling for the "Settings" footer link
- Settings vertical tab styling within /settings layout

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `packages/ui/src/components/sidebar.tsx` — shadcn Sidebar component; supports `collapsible="icon"` natively via `SidebarProvider` + `Sidebar collapsible` prop. No new library needed.
- `lucide-react` — Already installed; `PanelLeft`, `Settings`, `LayoutDashboard`, `CreditCard` all available for sidebar icons.
- `@clerk/tanstack-react-start` — `OrganizationSwitcher`, `UserButton` already imported in `AppSidebar.tsx`.
- `packages/ui/src/components/theme-toggle.tsx` — `ThemeToggle` already in sidebar footer; stays there.

### Established Patterns
- shadcn `collapsible="icon"` on `<Sidebar>` component causes nav items to show icon-only when collapsed — `<SidebarMenuButton>` handles this automatically when icons are direct children alongside text.
- `SidebarInset` wraps page content; removing the header from inside `SidebarInset` and adding a top bar outside `SidebarProvider` is the correct structural change.
- TanStack Router `<Outlet />` renders inside `LayoutShell`; the `_layout.settings/route.tsx` will need its own nested layout for the vertical sub-nav.

### Integration Points
- `apps/web/src/routes/__root.tsx` or `apps/web/src/routes/_layout.tsx` — Top bar insertion point (must be outside `SidebarProvider` or at least at the SidebarInset header level spanning full width)
- `apps/web/src/routes/_layout.settings/route.tsx` — Needs a vertical sub-nav layout added; currently just renders `<Outlet />`
- `AppSidebar.tsx` — All changes land here (header row redesign, nav structure, footer items)

</code_context>

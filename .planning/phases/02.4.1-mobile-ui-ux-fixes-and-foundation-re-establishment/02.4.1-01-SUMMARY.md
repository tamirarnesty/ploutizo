## What Was Built

### Task 1 â€” Viewport Containment (_layout.tsx)

`LayoutShell` now wraps `SidebarProvider` in an outer `div.flex.h-dvh.flex-col.overflow-hidden`. This div owns the viewport constraint. `SidebarProvider` becomes a flex child with `flex-col flex-1 min-h-0`, allowing it to shrink within the viewport. TopBar is now always visible and never scrolls with page content.

The key insight: adding `h-dvh` directly to `SidebarProvider` fights its internal `min-h-svh` class. Using a wrapper div we own avoids this conflict entirely.

### Task 2 â€” Sidebar Height Fix (AppSidebar.tsx)

Added `h-[calc(100svh-2.5rem)]` to the `<Sidebar>` element alongside existing `top-10`. Since the sidebar starts at `top: 2.5rem` (40px = `top-10`), the height must be `100svh - 2.5rem` to reach exactly the bottom of the viewport without overflow. `tailwind-merge` in `cn()` correctly overrides the base `h-svh` class.

### Task 2 â€” ThemeToggle in SidebarFooter (AppSidebar.tsx)

`SidebarFooter` now has `flex-row items-center justify-between px-2 py-1` with:
- `ThemeToggle` wrapped in `group-data-[collapsible=icon]:hidden` div â€” hidden when sidebar collapses to icon rail
- `SidebarTrigger` with `hidden md:flex` â€” shown only on desktop where icon-rail collapsing is relevant

Import added: `import { ThemeToggle } from "@ploutizo/ui/components/theme-toggle"`.

### Task 3 â€” Mobile Sidebar Auto-Close (AppSidebar.tsx)

`useSidebar()` context stored in a `useRef` following the `advanced-event-handler-refs` pattern. This gives `closeMobile` stable `[]` dependencies without subscribing `AppSidebar` to `isMobile` re-renders. `onClick={closeMobile}` added to all nav item Links and the Settings Link render prop.

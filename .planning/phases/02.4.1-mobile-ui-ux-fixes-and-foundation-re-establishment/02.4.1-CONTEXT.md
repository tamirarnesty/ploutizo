## Implementation Decisions

### D-01: Layout height constraint — fixes TopBar scrolling + document-level horizontal overflow

**File:** `apps/web/src/routes/_layout.tsx`

**Problem:** `SidebarProvider`'s wrapper div has `min-h-svh` by default, which allows it to grow beyond the viewport. When content overflows vertically, the TopBar (which lives inside this growing div) scrolls with the page. Similarly, wide content causes document-level horizontal scroll including the TopBar.

**Fix:** Add `h-dvh overflow-hidden` to `<SidebarProvider className="flex-col h-dvh overflow-hidden">`. This constrains the root container to exactly the dynamic viewport height and prevents any child from triggering document-level scroll. The `<main className="flex-1 overflow-auto p-6">` inside `SidebarInset` already handles per-page scrolling.

**Expected result:** TopBar is always fixed; only the content below it scrolls. No horizontal document-level scroll.

### D-02: Sidebar collapse button missing — sidebar overflows viewport

**File:** `apps/web/src/components/AppSidebar.tsx`

**Problem:** The sidebar component base class has `fixed inset-y-0 h-svh`. The current `className="top-10"` overrides `top` to `2.5rem`, but `h-svh` remains `100svh`. Result: sidebar runs from `2.5rem` to `2.5rem + 100svh`, extending `2.5rem` beyond the viewport bottom — the footer (and the collapse SidebarTrigger inside it) is clipped below the fold.

**Fix:** Change to `className="top-10 h-[calc(100svh-2.5rem)]"`. `tailwind-merge` (used in packages/ui `cn()`) will override `h-svh` → sidebar now runs from `2.5rem` to the bottom of the viewport exactly. Footer is visible.

**Expected result:** Collapse toggle appears at the bottom of the sidebar on desktop.

### D-06: ThemeToggle moves to Sidebar Footer, hidden when collapsed

**File:** `apps/web/src/components/AppSidebar.tsx`

**Target:** Sidebar Footer shows `[ThemeToggle] [SidebarTrigger]` side-by-side when expanded. When collapsed to icon-only, ThemeToggle is hidden; only SidebarTrigger remains.

**SidebarFooter** default renders as `flex flex-col`. Override to `flex-row items-center justify-between` so the two items sit in the same horizontal line.

**Hiding ThemeToggle on collapse:** Wrap it in `<div className="group-data-[collapsible=icon]:hidden">`. The sidebar's inner container has `data-collapsible="icon"` when collapsed with `collapsible="icon"`, and the sidebar wrapper carries the `group` class — so `group-data-[collapsible=icon]:hidden` correctly hides the wrapper when collapsed.

**Result SidebarFooter JSX:**
```tsx
<SidebarFooter className="flex-row items-center justify-between px-2 py-1">
  <div className="group-data-[collapsible=icon]:hidden">
    <ThemeToggle />
  </div>
  <SidebarTrigger />
</SidebarFooter>
```

**Import:** Add `ThemeToggle` import from `@ploutizo/ui/components/theme-toggle` to AppSidebar.

### D-07: 'd' key keyboard shortcut to cycle theme

**Files:** New file `apps/web/src/hooks/useThemeKeyboardShortcut.ts` + `apps/web/src/routes/_layout.tsx`

**Behavior:** Pressing `d` (no modifier keys) anywhere in the app cycles the theme (system → light → dark → system). Does NOT fire when focus is inside `INPUT`, `TEXTAREA`, `SELECT`, or a `contenteditable` element.

**Hook implementation:**
```ts
// apps/web/src/hooks/useThemeKeyboardShortcut.ts
import { useEffect } from "react"
import { useTheme } from "next-themes"

const cycleMap = { system: "light", light: "dark", dark: "system" } as const
type Theme = keyof typeof cycleMap

export function useThemeKeyboardShortcut() {
  const { theme, setTheme } = useTheme()
  useEffect(() => {
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key !== "d") return
      if (e.metaKey || e.ctrlKey || e.altKey || e.shiftKey) return
      const target = e.target as HTMLElement
      if (
        target.tagName === "INPUT" ||
        target.tagName === "TEXTAREA" ||
        target.tagName === "SELECT" ||
        target.isContentEditable
      ) return
      setTheme(cycleMap[(theme ?? "system") as Theme])
    }
    window.addEventListener("keydown", handleKeyDown)
    return () => window.removeEventListener("keydown", handleKeyDown)
  }, [theme, setTheme])
}
```

**Usage:** Call `useThemeKeyboardShortcut()` in `LayoutShell` (inside `_layout.tsx`).

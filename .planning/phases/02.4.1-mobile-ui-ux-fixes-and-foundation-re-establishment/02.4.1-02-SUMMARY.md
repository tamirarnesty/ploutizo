## What Was Built

### Task 1 — Sortable spacing (D-03, D-04)

Added `className="space-y-2"` to the `<Sortable>` component in both settings list views. Previously, sortable items had no vertical gap between them (they were flush). Now each item has 8px of space below it, matching standard list spacing in the app.

### Task 2 — TopBar redesign (D-05, D-08)

Rewrote TopBar.tsx to match the target layout:
- `OrganizationSwitcher` moved from standalone position (between AppLogo and ml-auto div) into the ml-auto div, to the LEFT of UserButton
- `ThemeToggle` import and usage removed — ThemeToggle now lives in the sidebar footer (added in plan 01)
- `showName` prop removed from `UserButton` — name label unnecessary in the new layout
- `UserButton` wrapped in `<div className="hidden md:block">` — hidden on mobile (sidebar footer provides profile access), visible on md+
- gap changed from gap-1 to gap-2 on the ml-auto div for better spacing with OrgSwitcher present

### Task 3 — Mobile-safe dialog width (D-09)

Both settings dialogs previously used only `max-w-*`, which on narrow screens (< 28rem) meant the dialog filled the full screen width with no margin. Added `w-[calc(100%-2rem)]` alongside the max-width:
- RuleDialog: `max-w-md` → `w-[calc(100%-2rem)] max-w-md`
- CategoryDialog: `max-w-sm` → `w-[calc(100%-2rem)] max-w-md` (also standardized from sm to md)

The `w-` property does not conflict with `max-w-` in tailwind-merge, so both survive — giving 1rem margin on each side on narrow screens.

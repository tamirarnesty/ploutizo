# Phase 02.3 Plan 04: CLAUDE.md Vercel Skills Guidelines Summary

**One-liner:** CLAUDE.md created at project root with explicit per-skill trigger conditions and SPA-scoped rule categories for all three Vercel skills.

## What Was Built

Created `CLAUDE.md` at the project root with:

1. **Stack section** — documents the full TanStack Start/Router SPA stack so agents understand the runtime context before writing any code
2. **Critical Constraints** — explicitly bans RSC patterns, server actions, `React.cache()`, raw `fetch()`, and `useState` for form fields; enforces project-specific patterns
3. **Testing constraint** — matches user's global CLAUDE.md preference (`pnpm test`, never `npx jest`)
4. **Vercel Skills section** — three skill entries, each with:
   - `**Trigger:**` imperative sentence defining when to apply
   - `**Action:**` exact steps (read SKILL.md, apply rules)
   - Inline rule summaries so agents don't need to read AGENTS.md for common cases

## Skill Sections Written

### vercel-react-best-practices
- Trigger: any task writing, refactoring, or reviewing React components/hooks in `apps/web`
- Lists all 8 SPA-applicable rule categories (RSC rules absent per 02.3-01 pruning)
- Each category has project-specific application notes (e.g., TanStack Query satisfies `client-swr-dedup`)

### vercel-composition-patterns
- Trigger: component API design, boolean prop additions, or reusable UI components
- Lists all 8 rules with React 19 note for `react19-no-forwardref`
- Notes that shadcn components in `packages/ui` may retain `forwardRef`

### web-design-guidelines
- Trigger: any UI-visible changes in `apps/web`
- Embeds live fetch URL: `https://raw.githubusercontent.com/vercel-labs/web-interface-guidelines/main/command.md`
- Specifies `file:line` terse output format
- Key rules inlined as reference (forms, focus, autoFocus, truncation, a11y, typography, motion)

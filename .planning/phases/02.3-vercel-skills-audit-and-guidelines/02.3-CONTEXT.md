**Gathered:** 2026-04-03

<domain>
## Phase Boundary

Audit the `apps/web` React codebase against three Vercel skills: `vercel-react-best-practices`, `vercel-composition-patterns`, and `web-design-guidelines`. Fix every violation found — no triage, no deferral. Then establish explicit GSD agent guidelines in `CLAUDE.md` (trigger conditions + applicable rules) so future agents automatically apply these skills without being asked.

`vercel-react-view-transitions` is installed but out of scope for this phase.

</domain>

<decisions>
## Implementation Decisions

### Next.js / RSC Rule Filtering

- **D-01:** Remove all Next.js/RSC-specific rules from `vercel-react-best-practices` that do not apply to a TanStack Start/Router + React SPA. This includes server components, server actions, `React.cache()`, `server-dedup-props`, and similar RSC-only patterns. Do not merely document them as N/A — delete them from the local skill file so future agents never encounter false positives.
- **D-02:** The SPA-applicable rule categories to prioritize (in order):
  1. **Eliminating Waterfalls (CRITICAL)** — `async-parallel`, `async-suspense-boundaries` etc., mapped to TanStack Query fetch patterns
  2. **Bundle Size (CRITICAL)** — barrel imports, dynamic imports for heavy components
  3. **Client-side Data Fetching (MEDIUM-HIGH)** — TanStack Query patterns, stale-while-revalidate, prefetching
  4. **Re-render Optimization (MEDIUM)** — memo, useCallback, stable references, especially in list-heavy components

### Fix Depth

- **D-03:** Audit + fix all violations in this single phase. Every violation found by running the three skills against `apps/web` must be fixed before the phase is complete. No known violations should be left open.
- **D-04:** No scope ceiling on refactor size. If fixing barrel imports touches 20+ files, fix all 20+. If adding Suspense boundaries requires restructuring route trees, do it. The full refactor is the deliverable.

### Guidelines Placement

- **D-05:** The resulting guidelines live in the root `CLAUDE.md` as a dedicated section. Every GSD agent reads `CLAUDE.md` automatically — no extra canonical_refs needed in future phases.
- **D-06:** Guidelines must specify **explicit trigger conditions** (not just descriptions). Format:
  ```
  ## Vercel Skills — When to Apply

  ### vercel-react-best-practices
  Trigger: any task that writes, refactors, or reviews React components in apps/web.
  Scope: [list of applicable rule categories after Next.js pruning]

  ### vercel-composition-patterns
  Trigger: any task that designs or refactors component APIs, adds boolean props, or builds reusable UI.
  Scope: [applicable rules]

  ### web-design-guidelines
  Trigger: any task that adds or modifies UI visible to the user (pages, dialogs, forms, layouts).
  Scope: fetches fresh guidelines from https://... each run
  ```

### Claude's Discretion

- Exact order in which audit runs are structured within the plan (single-pass vs. per-skill passes)
- Whether to group fixes by skill or by file/component
- Minor formatting of the CLAUDE.md guidelines section

</decisions>

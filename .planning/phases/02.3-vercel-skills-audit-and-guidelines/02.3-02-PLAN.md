  <action>
    The current code at lines 54-57 calls `setLocalCategories(categories)` during the render body:
    ```tsx
    const [localCategories, setLocalCategories] = useState<Array<Category>>([])
    if (!catLoading && localCategories.length === 0 && categories.length > 0) {
      setLocalCategories(categories)
    }
    ```
    This triggers a double-render every time the component first loads data. React calls the component twice: once to render with empty array, once after the state update. This is the `rerender-derived-state-no-effect` violation.

    Fix: Use a `useRef` flag to initialize local state exactly once after data loads, without triggering a re-render cascade.

    Add `useRef` to the React import at the top of the file (it is already imported from "react" via `useState` — just add `useRef`).

    Replace the problematic block:
    ```tsx
    // Locally manage category order for optimistic reorder UX
    const [localCategories, setLocalCategories] = useState<Array<Category>>([])
    if (!catLoading && localCategories.length === 0 && categories.length > 0) {
      setLocalCategories(categories)
    }
    ```
    with:
    ```tsx
    // Locally manage category order for optimistic reorder UX
    const [localCategories, setLocalCategories] = useState<Array<Category>>([])
    const categoriesInitialized = useRef(false)

    // Initialize from server data exactly once on first successful load.
    // useRef flag prevents re-initialization on subsequent server invalidations,
    // which would wipe in-flight optimistic drag order.
    useEffect(() => {
      if (!categoriesInitialized.current && !catLoading && categories.length > 0) {
        categoriesInitialized.current = true
        setLocalCategories(categories)
      }
    }, [catLoading, categories])
    ```

    Add `useEffect` to the React import if not already there. The full import line should become:
    ```tsx
    import { useState, useRef, useEffect } from "react"
    ```

    IMPORTANT: Do NOT use a plain `useEffect` with `[categories]` as the only dependency — this would re-sync on every mutation invalidation and wipe the user's drag order. The `categoriesInitialized.current = true` flag is essential to ensure initialization happens exactly once.

    IMPORTANT: The research warns against a `useEffect` with `categories` as dependency WITHOUT the ref guard because it re-fires on refetch. The ref guard in this implementation is specifically designed to prevent that. This is different from the naive fix.
  </action>

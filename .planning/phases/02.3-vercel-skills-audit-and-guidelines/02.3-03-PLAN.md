    **Pre-identified violations to fix (verify each still exists, then fix):**

    **1. LucideIconPicker.tsx — autoFocus anti-pattern:**
    The `<Input autoFocus ...>` in LucideIconPicker is flagged by web-design-guidelines as an anti-pattern. `autoFocus` should only be used when:
    - The input is the primary purpose of the page/dialog
    - The dialog was explicitly opened by user action
    LucideIconPicker is a picker component opened from a dialog — user action. This is a valid use case. Add a comment justifying it:
    ```tsx
    <Input
      autoFocus  // Picker opened by user action — autoFocus on search is intentional and expected
      ...
    ```
    If the guidelines fetched from the URL explicitly prohibit autoFocus even in user-initiated dialogs, remove the attribute instead. Follow the live guidelines, not this plan's assumption.

    **2. AccountForm.tsx — missing autocomplete attributes:**
    Add appropriate `autocomplete` attributes to the name and institution inputs. Read the current AccountForm.tsx (already modified in Plan 02) and add:
    - Account name input: `autoComplete="off"` (account names are custom, not browser-autocomplete candidates)
    - Institution input: `autoComplete="organization"` (browser can suggest known financial institution names)
    - Last four digits input: `autoComplete="off"` (security-relevant partial card number)

    **3. Flex children with truncation — min-w-0:**
    Any flex child that renders text which could be long (category names, account names, tag names in lists) should have `min-w-0` applied to prevent flex children from overflowing their container. Check these patterns:
    - `className="flex-1 text-sm"` — add `min-w-0` if text could overflow: `className="min-w-0 flex-1 truncate text-sm"`
    - `className="flex items-center gap-..."` parent containers where children render truncated text

    Specifically check:
    - CategoriesSettings.tsx category list items: `<span className="flex-1 text-sm">{cat.name}</span>` — should be `<span className="min-w-0 flex-1 truncate text-sm">{cat.name}</span>`

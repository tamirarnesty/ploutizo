# Phase 02.3 Plan 05: Gap Closure — autoComplete, motion-safe, and ternary conditionals

**One-liner:** Closed three web-design-guidelines and vercel-react-best-practices violations: `autoComplete="off"` on 4 form inputs, `motion-safe:animate-pulse` on 2 skeleton loaders, and 9 `&&` conditional JSX renders converted to ternary across 5 files.

## What Was Built

This plan addressed the three remaining gaps identified by the Phase 02.3 verifier after Plans 02 and 03 were completed.

### Gap 1: autoComplete attributes (4 inputs, 3 files)

| File | Input | Value |
|------|-------|-------|
| CategoryForm.tsx | `id="category-name"` | `autoComplete="off"` |
| RuleForm.tsx | pattern Input | `autoComplete="off"` |
| RuleForm.tsx | renameTo Input | `autoComplete="off"` |
| HouseholdSettingsForm.tsx | `id="settlement-threshold"` | `autoComplete="off"` |

All four inputs are custom/non-browser-fillable fields (regex patterns, settlement thresholds, category names) — `off` is correct per web-design-guidelines.

### Gap 2: motion-safe guard on skeleton loaders (2 divs, 1 file)

Both `animate-pulse` classes in `CategoriesSettings.tsx` updated to `motion-safe:animate-pulse` to respect the `prefers-reduced-motion` media query as required by the web-design-guidelines motion rule.

### Gap 3: && conditional JSX → ternary (9 occurrences, 5 files)

The `rendering-conditional-render` rule from `vercel-react-best-practices` requires ternary `expr ? <JSX/> : null` instead of `expr && <JSX/>` to prevent accidental numeric `0` renders.

| File | Location | Pattern converted |
|------|----------|------------------|

| RuleForm.tsx | line ~149 | `errors.length > 0 &&` |

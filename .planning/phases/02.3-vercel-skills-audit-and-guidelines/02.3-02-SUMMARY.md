## What Was Done

### Task 1: AccountForm parallel-fetch waterfall (async-parallel rule)

**Problem:** `AccountForm` called `useGetAccountMembers`, early-returned a spinner on load, then rendered `AccountFormInner` which called `useGetOrgMembers`. This created a sequential waterfall — `useGetOrgMembers` could not fire until `useGetAccountMembers` resolved.

**Fix:** Both queries lifted into `AccountForm` (outer component). `AccountFormInner` now receives `orgMembers` as a prop instead of calling the hook itself. Combined loading gate: `membersLoading || orgLoading`.

**Key change:** Added `OrgMember` type import from `@ploutizo/types`, added `orgMembers: Array<OrgMember>` to `AccountFormInnerProps`.

### Task 2: CategoriesSettings setState-during-render (rerender-derived-state-no-effect)

**Problem:** Lines 54-57 called `setLocalCategories(categories)` directly during the render body, triggering a double-render cycle on every component mount when data loads.

**Fix:** Replaced the in-render conditional assignment with a `useEffect` + `useRef` guard pattern. The `categoriesInitialized` ref ensures initialization fires exactly once — not on subsequent TanStack Query invalidations that would wipe in-flight drag order.

**Key change:** Added `useRef` and `useEffect` to React imports.

### Task 3: QueryClient staleTime (client-swr-dedup rule)

**Verification:** `staleTime` was already `1000 * 60` (60 seconds) — no functional change needed.

**Documentation:** Updated comment to explicitly reference the `client-swr-dedup` rule and explain TanStack Query deduplication semantics for future agents.

## Decisions Made

1. **AccountFormInner receives orgMembers as prop** — Lifting both queries to the parent eliminates the waterfall while preserving the shell/inner component structure established in Phase 02.1. The inner component API now explicitly declares its data dependencies.

2. **useRef guard for one-time initialization** — A plain `useEffect([categories])` without the ref would re-initialize on every TanStack Query refetch/invalidation, wiping the user's drag-ordered `localCategories`. The ref guard is essential to prevent this.

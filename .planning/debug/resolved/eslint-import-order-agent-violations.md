## Resolution

root_cause: The .claude/settings.local.json PostToolUse hook only runs Prettier (pnpm format) after Write/Edit tool calls. No ESLint --fix hook exists anywhere. The @tanstack/eslint-config enforces import/order as an error, but since ESLint is never auto-run after file writes, import ordering violations accumulate silently. No CLAUDE.md exists to give agents explicit instructions, and no lint:fix scripts existed in package.json files.
fix: (1) Added "lint:fix": "eslint --fix" to apps/web/package.json and packages/ui/package.json. (2) Created .claude/hooks/gsd-lint-fix.js — a PostToolUse hook that extracts file_path from stdin JSON and runs eslint --fix from the correct package directory for .ts/.tsx files. (3) Registered the new hook in settings.local.json PostToolUse block alongside the existing Prettier hook. (4) Fixed all existing import/order violations across 8 files.
verification: pnpm lint in apps/web shows 0 import/order violations. Hook tested end-to-end — exits 0 cleanly.
